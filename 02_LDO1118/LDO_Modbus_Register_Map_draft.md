# LDO. Карта регистров Modbus RTU

## Общие сведения

Документ описывает карту Holding-регистров Modbus RTU встроенного программного обеспечения модуля релейных выходов **LDO1128**.

Документ подготовлен по исходному коду проекта **LDO_Rev3**.

Исходные файлы, использованные при подготовке:

```text
app/app.cpp
app/ldo_board.cpp
app/ldo_board.hpp
Libs/IGAS_mb.h
Libs/IGAS_mb.cpp
```

---

## Параметры обмена

| Параметр | Значение |
|---|---|
| Протокол обмена | Modbus RTU |
| Физический интерфейс | RS-485, двухпроводная линия |
| Режим устройства | Slave |
| Скорость обмена | 9600 бод |
| Формат кадра UART | 8N1 |
| Чётность | Нет |
| Стоп-биты | 1 |
| Порядок байт в регистрах | Big-endian, старший байт передаётся первым |
| Максимальное количество регистров за один запрос чтения | 15 |
| Максимальное количество регистров за один запрос записи | 15 |
| Диапазон допустимых адресов Holding-регистров на уровне транспорта | 0...1000 |
| Адресация регистров в документе | Нулевая, как в Modbus PDU |

> Примечание: если Modbus-клиент использует отображение вида `40001`, то регистр `HR 0` может отображаться в клиенте как `40001`, регистр `HR 1` — как `40002` и так далее.

## Поддерживаемые функции Modbus

| Код функции | Название | Назначение |
|---|---|---|
| `0x03` | Read Holding Registers | Чтение Holding-регистров |
| `0x06` | Write Single Register | Запись одного Holding-регистра |
| `0x10` | Write Multiple Registers | Запись нескольких Holding-регистров |

## Адрес устройства

Modbus-адрес устройства считывается с аппаратного адресного переключателя при запуске модуля.

| Вход микроконтроллера | Вес бита адреса |
|---|---|
| `PC5` | `0x01` |
| `PC4` | `0x02` |
| `PC3` | `0x04` |
| `PC2` | `0x08` |
| `PC1` | `0x10` |

Если все входы адресного переключателя равны нулю, прошивка использует адрес `1`.

## Карта Holding-регистров

| Адрес регистра | Отображение 4xxxx | Тип | Доступ | Значение / диапазон | Назначение |
|---:|---:|---|---|---|---|
| `0` | `40001` | `uint16` | `R/W` | `0x0000...0x00FF` | Уставка и чтение состояния релейных выходов. При записи используются младшие восемь бит. При чтении возвращается текущее физически применённое состояние выходов. |
| `1` | `40002` | `uint16` | `R` | `0x0000...0x00FF` | Чтение текущего физически применённого состояния релейных выходов. |
| `3` | `40004` | `uint16` | `R` | `0x0000...0x00FF` | Код ошибки устройства. В текущей версии прошивки код ошибки инициализируется значением `0x00`. Перечень ненулевых кодов ошибок в исходном коде не задан. |
| `580` | `40581` | `uint16` | `R/W` | ключ: `20118` | Сервисный режим. При записи значения `20118` устанавливается флаг сервисного режима. При чтении возвращается состояние флага сервисного режима: `0` — не активен, `1` — активен. |
| `590` | `40591` | `uint16` | `R` | `0` | Резерв сервисных данных. |
| `591` | `40592` | `uint16` | `R` | `0` | Резерв сервисных данных. |
| `592` | `40593` | `uint16` | `R` | `0` | Резерв сервисных данных. |
| `593` | `40594` | `uint16` | `R` | `0` | Резерв сервисных данных. |
| `594` | `40595` | `uint16` | `R/W` | `0` / `1` | Программная перезагрузка. При записи любого значения устанавливается флаг перезагрузки. Перезагрузка выполняется через watchdog. При чтении возвращается текущее состояние флага. |
| `900` | `40901` | `uint16` | `R/W` | `0` / `1` | Отложенный переход в bootloader. При записи любого значения запускается отложенный переход в bootloader. При чтении возвращается `1`, если переход запрошен, и `0`, если переход не запрошен. |
| прочие адреса `0...1000` | — | `int16` | `R/W` на уровне транспорта | `-1` при чтении | Неподдерживаемые / неиспользуемые регистры. При чтении прикладной обработчик возвращает `-1`. Запись на уровне Modbus может быть принята, но прикладного действия не выполняет. |
| адреса `>=1001` | — | — | — | Modbus exception | Недопустимый диапазон адресов. |

## Регистр состояния релейных выходов

Релейные выходы управляются битовой маской через регистр `HR 0`.

| Бит | Маска | Логический выход | Состояние при `0` | Состояние при `1` |
|---:|---:|---|---|---|
| `0` | `0x01` | `DO1` | Реле выключено | Реле включено |
| `1` | `0x02` | `DO2` | Реле выключено | Реле включено |
| `2` | `0x04` | `DO3` | Реле выключено | Реле включено |
| `3` | `0x08` | `DO4` | Реле выключено | Реле включено |
| `4` | `0x10` | `DO5` | Реле выключено | Реле включено |
| `5` | `0x20` | `DO6` | Реле выключено | Реле включено |
| `6` | `0x40` | `DO7` | Реле выключено | Реле включено |
| `7` | `0x80` | `DO8` | Реле выключено | Реле включено |

При записи в `HR 0` используется только младший байт значения. Старший байт игнорируется.

Примеры значений:

| Значение `HR 0` | Состояние |
|---:|---|
| `0x0000` | Все релейные выходы выключены |
| `0x0001` | Включён только `DO1` |
| `0x0003` | Включены `DO1` и `DO2` |
| `0x00FF` | Включены все релейные выходы |

## Особенности применения команд

### Запись состояния релейных выходов

Для изменения состояния релейных выходов необходимо записать битовую маску в `HR 0`.

Поддерживается запись функциями:

```text
0x06 Write Single Register
0x10 Write Multiple Registers
```

Физическое применение нового состояния выполняется в периодической задаче прошивки с периодом около `100 мс`.

### Чтение состояния релейных выходов

Текущее физически применённое состояние можно читать из регистров:

```text
HR 0
HR 1
```

### Потеря связи

Если в течение `10000 мс` не было корректного Modbus-запроса к устройству, прошивка сбрасывает команду реле в безопасное состояние:

```text
newState = 0x00
```

После этого релейные выходы переводятся в выключенное состояние при следующем применении состояния выходов.

### Программная перезагрузка

Для программной перезагрузки необходимо записать любое значение в `HR 594`.

После записи прошивка устанавливает флаг перезагрузки и выполняет сброс микроконтроллера через watchdog.

### Переход в bootloader

Для перехода в bootloader необходимо записать любое значение в `HR 900`.

Переход выполняется с задержкой около `50 мс`, чтобы Modbus-ответ успел уйти в линию RS-485.

Стартовый адрес bootloader в текущем исходном коде:

```text
0x7000
```

## Исключения Modbus

| Условие | Ответ |
|---|---|
| Неподдерживаемый код функции | Exception `0x01` — Illegal Function |
| Начальный адрес регистра вне диапазона транспорта | Exception `0x02` — Illegal Data Address |
| Количество регистров меньше `1` или больше `15` | Exception `0x03` — Illegal Data Value |
| Запрос выходит за верхнюю границу диапазона транспорта | Exception `0x02` — Illegal Data Address |

## Требует подтверждения перед финальной сдачей

Перед выпуском финальной версии документа необходимо подтвердить следующие пункты:

- точную версию прошивки, к которой относится карта регистров;
- дату релиза;
- нужно ли показывать сервисный ключ регистра `HR 580` во внешней документации или оставить этот регистр только во внутреннем документе;
- нужно ли показывать регистр `HR 900` перехода в bootloader во внешней документации;
- соответствие логических выходов `DO1...DO8` фактической маркировке клемм на корпусе LDO1128;
- требуются ли отдельные регистры идентификатора устройства, версии ПО и аппаратной ревизии;
- перечень кодов ошибок для регистра `HR 3`, если ненулевые коды ошибок должны поддерживаться;
- считать ли неиспользуемые адреса публично “резервными” или исключить их из пользовательской документации.

## Краткий пример использования

### Включить выходы `DO1` и `DO3`

Записать в `HR 0` значение:

```text
0x0005
```

Потому что:

```text
DO1 = 0x01
DO3 = 0x04
0x01 | 0x04 = 0x05
```

### Выключить все выходы

Записать в `HR 0` значение:

```text
0x0000
```

### Прочитать состояние выходов

Прочитать один регистр, начиная с адреса:

```text
HR 1
```

или прочитать `HR 0`, если требуется использовать основной регистр управления / состояния.
