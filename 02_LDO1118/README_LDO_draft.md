# LDO Firmware

Документация по встроенному программному обеспечению модуля **LDO**.

Прошивка предназначена для работы на модуле дискретных выходов LDO и реализует управление релейными выходами, обмен по **Modbus RTU** через интерфейс **RS-485**, контроль связи с ведущим устройством и сервисные функции перезагрузки / перехода в bootloader.

---

## Назначение

Прошивка выполняет следующие функции:

- инициализация аппаратных ресурсов платы LDO;
- считывание Modbus-адреса с аппаратного адресного переключателя;
- обмен с ведущим устройством по Modbus RTU через RS-485;
- обработка Holding-регистров Modbus;
- управление восемью релейными выходами;
- сброс релейных выходов в безопасное состояние при потере связи;
- индикация состояния связи светодиодом статуса;
- программная перезагрузка через watchdog;
- отложенный переход в bootloader по команде Modbus;
- обслуживание watchdog в основном цикле программы.

## Состав исходного проекта

- `main.cpp` — точка входа программы. Выполняет базовую инициализацию, запуск системного таймера и передачу управления функциям `setup()` / `loop()`.
- `app/` — прикладной уровень прошивки. Содержит основную логику LDO, обработку Modbus-регистров, контроль связи, управление релейными выходами и сервисные команды.
- `app/ldo_board.*` — аппаратная карта платы LDO. Содержит привязку сигналов к портам ATmega328P, управление RS-485, светодиодом статуса, адресным переключателем и релейными выходами.
- `hal/` — минимальный аппаратно-зависимый слой совместимости: GPIO, Serial, SPI, системный тик.
- `Libs/` — локальные библиотеки проекта, включая реализацию Modbus RTU.
- `LDO_Rev3.cppproj` — проект Microchip Studio / Atmel Studio для сборки прошивки.
- `Doxyfile` — конфигурация генерации документации Doxygen.

## Целевая аппаратная платформа

| Параметр | Значение |
|---|---|
| Модуль | LDO |
| Тип устройства | Модуль дискретных / релейных выходов |
| Микроконтроллер | ATmega328P |
| Частота тактирования | `F_CPU = 16000000UL` |
| Интерфейс связи | RS-485 |
| Протокол | Modbus RTU |
| Скорость обмена | `9600` бод |
| Формат кадра | `SERIAL_8N1` |
| Количество релейных выходов | 8 |
| Активная аппаратная ревизия в исходном коде | `BOARD_NEW_RS485_PD2` |
| Линия управления RS-485 для активной ревизии | `PD2` |
| Светодиод статуса | `PC0` |
| Watchdog | включён, таймаут `WDTO_1S` |
| Стартовый адрес bootloader | `0x7000` байт |

Активная аппаратная ревизия выбирается в файле:

```text
app/ldo_board.cpp
```

Текущая настройка:

```cpp
static const ldo_board::BoardRevision LDO_BOARD_REVISION =
    ldo_board::BOARD_NEW_RS485_PD2;
```

После изменения аппаратной ревизии требуется пересборка прошивки.

## Адрес устройства Modbus

Modbus-адрес устройства считывается с аппаратного адресного переключателя.

Соответствие входов и битов адреса:

| Вход микроконтроллера | Бит адреса |
|---|---|
| `PC5` | bit 0 |
| `PC4` | bit 1 |
| `PC3` | bit 2 |
| `PC2` | bit 3 |
| `PC1` | bit 4 |

Если все входы адресного переключателя равны нулю, используется адрес `1`.

## Основная логика работы

После сброса микроконтроллера выполняется функция `main()`:

1. запрещаются прерывания;
2. инициализируется системный миллисекундный тик `hal::tick_init()`;
3. разрешаются прерывания;
4. вызывается `setup()`;
5. управление передаётся бесконечному циклу `loop()`.

Функция `setup()` выполняет:

- сброс и настройку watchdog;
- инициализацию аппаратных линий платы LDO;
- чтение Modbus-адреса с адресного переключателя;
- инициализацию программных таймеров;
- инициализацию Modbus RTU.

Функция `loop()` выполняет:

- обновление текущего времени;
- периодическую задачу с периодом около `100` мс;
- периодическую задачу с периодом около `500` мс;
- обслуживание Modbus RTU;
- сброс watchdog.

---

## Периодические задачи

### Задача `loop_100()`

Период выполнения: около `100` мс.

Выполняет:

- применение нового состояния релейных выходов;
- программную перезагрузку через watchdog при установленном флаге перезагрузки;
- отложенный переход в bootloader после получения соответствующей команды.

### Задача `loop_500()`

Период выполнения: около `500` мс.

Выполняет:

- контроль таймаута связи с ведущим устройством;
- сброс требуемого состояния релейных выходов при потере связи;
- мигание светодиодом статуса при отсутствии связи.

Таймаут потери связи:

```cpp
LOST_CONNECTION_TIME_MS = 10000UL
```

Если в течение этого времени не было корректного обращения по Modbus, прошивка сбрасывает команду реле в безопасное состояние.

## Карта Holding-регистров Modbus

| Регистр | Доступ | Назначение | Примечание |
|---:|---|---|---|
| `HR 0` | чтение / запись | Управление релейными выходами | При записи задаёт новое состояние выходов. При чтении возвращает текущее применённое состояние. |
| `HR 1` | чтение | Текущее состояние релейных выходов | Дублирующий регистр чтения состояния. |
| `HR 3` | чтение | Код ошибки устройства | Возвращает значение `errorByte`. |
| `HR 580` | чтение / запись | Сервисный режим | Запись сервисного ключа `20118` устанавливает флаг сервисного режима. |
| `HR 590` | чтение | Резерв сервисных данных | Возвращает `0`. |
| `HR 591` | чтение | Резерв сервисных данных | Возвращает `0`. |
| `HR 592` | чтение | Резерв сервисных данных | Возвращает `0`. |
| `HR 593` | чтение | Резерв сервисных данных | Возвращает `0`. |
| `HR 594` | чтение / запись | Программная перезагрузка | При записи устанавливает флаг перезагрузки через watchdog. |
| `HR 900` | чтение / запись | Переход в bootloader | При записи запускает отложенный переход в bootloader. |

Для неподдерживаемых регистров возвращается значение `-1`.

## Управление релейными выходами

Состояние релейных выходов задаётся битовой маской через регистр `HR 0`.

Для активной ревизии `BOARD_NEW_RS485_PD2` используется следующее соответствие битов физическим линиям:

| Бит состояния | Линия микроконтроллера |
|---:|---|
| bit 0 | `PB2` |
| bit 1 | `PB1` |
| bit 2 | `PB0` |
| bit 3 | `PD7` |
| bit 4 | `PD6` |
| bit 5 | `PD5` |
| bit 6 | `PD4` |
| bit 7 | `PD3` |

При потере связи значение требуемого состояния выходов сбрасывается в `0x00`.

---

## Индикация состояния

Светодиод статуса подключён к линии `PC0`.

Логика индикации:

- при получении корректного обращения по Modbus светодиод включается;
- при отсутствии связи светодиод мигает с периодической обработкой в задаче `loop_500()`.

## Bootloader

Прошивка поддерживает программный переход в bootloader по команде Modbus через регистр `HR 900`.

Стартовый адрес bootloader:

```cpp
BOOT_START_ADDR_BYTES = 0x7000U
```

Перед переходом в bootloader прошивка:

- запрещает прерывания;
- отключает watchdog;
- отключает UART, SPI и TWI, если соответствующие регистры доступны;
- восстанавливает стек;
- переключает таблицу векторов прерываний в boot-секцию;
- выполняет прямой переход на адрес bootloader.

Значение стартового адреса должно соответствовать фактическим fuse-настройкам `BOOTSZ` и сборке bootloader.

## Минимальная проверка работоспособности

После прошивки рекомендуется выполнить следующие проверки:

| Проверка | Действие | Ожидаемый результат |
|---|---|---|
| Запуск устройства | Подать питание на модуль | Модуль запускается, watchdog не вызывает циклический сброс |
| Проверка связи | Выполнить чтение доступного Holding-регистра по Modbus RTU | Устройство отвечает без ошибки обмена |
| Проверка адреса | Установить адрес переключателем и выполнить опрос | Устройство отвечает на установленный адрес |
| Проверка состояния выходов | Прочитать `HR 1` | Возвращается текущее состояние выходов |
| Проверка управления выходами | Записать битовую маску в `HR 0` | Соответствующие релейные выходы переключаются |
| Проверка сброса выходов | Прекратить обмен более чем на `10000` мс | Требуемое состояние выходов сбрасывается в `0x00` |
| Проверка перезагрузки | Записать значение в `HR 594` | Устройство перезагружается через watchdog |
| Проверка bootloader | Записать значение в `HR 900` | Устройство выполняет переход в bootloader |
