# Шаблон канонического outline для Board Guide / пояснительной записки

ℹ️ Этот шаблон — каноническая основа для `Stage 3`.

ℹ️ На конкретном проекте его нужно сокращать и адаптировать под реальный состав изделия.

ℹ️ `outline.md` должен быть **рабочим читаемым документом**, а не россыпью заголовков.

Обязательные правила оформления:

* в начале файла — нумерованное оглавление;
* заголовки в теле документа повторяют ту же нумерацию;
* крупные разделы отделяются `---`;
* для крупных функциональных глав заранее закладываются одинаковые логические слои:
  * назначение;
  * аппаратная реализация;
  * ключевые характеристики;
  * возможности и сценарии применения;
  * ограничения и последствия модификации.
