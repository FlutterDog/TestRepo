## 1. Назначение релиза

Настоящий релиз содержит исходный проект, сопроводительную документацию и скомпилированный файл прошивки встроенного программного обеспечения модуля LDO1128.

Прошивка предназначена для работы на модуле релейных выходов LDO1128 и обеспечивает:
- инициализацию аппаратных ресурсов платы;
- считывание адреса модуля с аппаратного адресного переключателя;
- обмен с ведущим устройством по Modbus RTU через RS-485 / X2X;
- обработку Holding-регистров Modbus;
- управление восемью релейными выходами;
- контроль наличия связи с ведущим устройством;
- сброс релейных выходов при потере связи;
- индикацию состояния связи;
- программную перезагрузку;
- отложенный переход в bootloader.

## 2. Сведения об изделии

- Наименование изделия: модуль релейных выходов LDO1128
- Линейка изделия: Lorentz
- Назначение изделия: коммутация внешних цепей с помощью выходных реле
- Исполнение: пластиковый корпус для установки на DIN-рейку
- Количество цифровых выходов: 8
- Тип выходов: релейные выходы, сухой контакт
- Группы контактов реле: нормально открытая и нормально закрытая
- Номинальное управление нагрузкой: до 250 В переменного тока / 30 В постоянного тока
- Коммутируемый ток одного выхода: 2 А
- Интерфейс управления: X2X / RS-485
- Протокол обмена прошивки: Modbus RTU
- Адресация: аппаратный адресный переключатель

## 4. Параметры обмена по умолчанию

Протокол: Modbus RTU
Физический интерфейс: RS-485, двухпроводная схема
Скорость обмена: 9600 бод
Формат кадра: 8N1
Четность: нет
Стоп-бит: 1
Адрес по умолчанию при нулевом положении адресного переключателя: 1
Поддерживаемые функции Modbus в библиотеке обмена:
- 0x03 Read Holding Registers;
- 0x06 Write Single Register;
- 0x10 Write Multiple Registers.

## 5. Карта Holding-регистров прошивки

HR 0:
- доступ: чтение / запись;
- назначение при записи: установка требуемого состояния релейных выходов;
- назначение при чтении: чтение текущего физически примененного состояния релейных выходов;
- формат: битовая маска, младшие 8 бит.

HR 1:
- доступ: чтение;
- назначение: чтение текущего физически примененного состояния релейных выходов;
- формат: битовая маска, младшие 8 бит.

HR 3:
- доступ: чтение;
- назначение: код ошибки устройства.

HR 580:
- доступ: запись / чтение;
- назначение: вход в сервисный режим;
- сервисный ключ: 20118;
- результат чтения: состояние сервисного режима.

HR 590...593:
- доступ: чтение;
- назначение: резерв сервисных данных;
- текущее значение: 0.

HR 594:
- доступ: запись / чтение;
- назначение: программная перезагрузка через watchdog.

HR 900:
- доступ: запись / чтение;
- назначение: отложенный переход в bootloader.

Прочие регистры:
- значение ответа в текущей реализации: -1;
- назначение: недопустимые или неиспользуемые регистры.

## 6. Соответствие битов релейным выходам для активной аппаратной ревизии

Активная ревизия: BOARD_NEW_RS485_PD2

bit 0 -> PB2
bit 1 -> PB1
bit 2 -> PB0
bit 3 -> PD7
bit 4 -> PD6
bit 5 -> PD5
bit 6 -> PD4
bit 7 -> PD3

Примечание: в исходном коде также сохранено описание старой аппаратной ревизии BOARD_OLD_RS485_PB2. Для применения старой ревизии требуется изменение константы LDO_BOARD_REVISION и пересборка прошивки.

## 11. Ограничения и условия применения

Прошивка предназначена для аппаратной платформы LDO на микроконтроллере ATmega328P.
Перед применением на другой аппаратной ревизии необходимо проверить карту выводов, линию управления RS-485, fuse-настройки bootloader и соответствие стартового адреса bootloader значению 0x7000.

Перед загрузкой в изделие необходимо убедиться, что:
- выбран файл LDO_Rev3.hex из папки релиза;
- контрольная сумма файла соответствует указанной в настоящей описи;
- аппаратная ревизия платы соответствует BOARD_NEW_RS485_PD2 либо прошивка пересобрана под требуемую ревизию;
- параметры обмена Modbus RTU соответствуют настройкам ведущего устройства;
- адрес модуля установлен аппаратным адресным переключателем.
