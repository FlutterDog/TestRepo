## LDO. Инструкция по загрузке прошивки

### Назначение

Документ описывает порядок загрузки встроенного программного обеспечения в модуль релейных выходов **LDO1128**.

Инструкция предназначена для подготовки, загрузки и первичной проверки прошивки проекта **LDO_Rev3** на целевой аппаратной платформе модуля LDO. В качестве основного файла прошивки используется файл:

```text
LDO_Rev3.hex
```

Прошивка предназначена для модуля LDO1128 с микроконтроллером **ATmega328P** и тактовой частотой **16 MHz**. Обмен с ведущим устройством выполняется по интерфейсу **RS-485** в режиме **Modbus RTU**. В текущей версии проекта используется аппаратная ревизия платы с управлением направлением RS-485 по линии **PD2**.

---

Файл `LDO_Rev3.hex` является основным исполняемым файлом прошивки и используется для загрузки в микроконтроллер. Исходный проект `LDO_Rev3.cppproj` предназначен для сборки прошивки в Atmel Studio / Microchip Studio. Файл `README.md` и Doxygen-документация используются как справочные материалы по составу проекта, карте регистров и внутренней структуре прошивки.

### Целевая аппаратная платформа

Прошивка предназначена для модуля **LDO1128** — модуля релейных выходов серии Lorentz.

Основные аппаратные параметры, относящиеся к загрузке и первичной проверке прошивки:

| Параметр                                          | Значение                                              |
| ------------------------------------------------- | ----------------------------------------------------- |
| Модуль                                            | LDO1128                                               |
| Назначение модуля                                 | Управление внешними цепями с помощью релейных выходов |
| Микроконтроллер                                   | ATmega328P                                            |
| Тактовая частота                                  | 16 MHz                                                |
| Количество релейных выходов                       | 8                                                     |
| Интерфейс связи                                   | RS-485                                                |
| Протокол обмена                                   | Modbus RTU                                            |
| Скорость обмена по умолчанию                      | 9600 бод                                              |
| Формат кадра                                      | 8N1                                                   |
| Адрес устройства                                  | Задаётся аппаратным адресным переключателем           |
| Линия управления RS-485 DE/RE для текущей ревизии | PD2                                                   |
| Файл прошивки                                     | `LDO_Rev3.hex`                                        |

В прошивке предусмотрены две аппаратные карты платы:

| Ревизия платы         | Линия управления RS-485 | Линии релейных выходов   |
| --------------------- | ----------------------- | ------------------------ |
| `BOARD_OLD_RS485_PB2` | PB2                     | PD2...PD7, PB0, PB1      |
| `BOARD_NEW_RS485_PD2` | PD2                     | PD3...PD7, PB0, PB1, PB2 |

В текущем проекте активна ревизия:

```cpp
BOARD_NEW_RS485_PD2
```

Для этой ревизии линия **PD2** используется для управления направлением RS-485-трансивера, поэтому при загрузке прошивки и последующей проверке необходимо учитывать, что прикладной код переключает RS-485 между режимами приёма и передачи программно.

### Необходимые средства

Для загрузки прошивки требуются:

* персональный компьютер с установленной средой Atmel Studio / Microchip Studio либо другим ПО, поддерживающим загрузку HEX-файлов в ATmega328P;
* файл прошивки `LDO_Rev3.hex`;
* модуль LDO1128;
* источник питания модуля;
* программатор для ATmega328P либо штатный загрузчик, если он установлен в микроконтроллере;
* кабель подключения программатора или интерфейса загрузчика;
* преобразователь USB-RS485 для проверки связи после прошивки;
* программа для работы с Modbus RTU, например Modbus Poll, QModMaster или аналогичный инструмент;
* нагрузка, индикатор, мультиметр или контрольный стенд для проверки релейных выходов.

Для проверки работы по RS-485 должны быть доступны линии:

```text
A+
A-
GND
+24 В
```

Адрес устройства задаётся аппаратным DIP-переключателем. В прошивке используются входы **PC1...PC5**. Если все биты адресного переключателя находятся в нулевом состоянии, прошивка использует адрес **1**.

### Подготовка к прошивке

Перед загрузкой прошивки необходимо проверить:

* соответствие модуля аппаратной ревизии, для которой собран проект;
* наличие файла `LDO_Rev3.hex` в составе релизного комплекта;
* исправность питания модуля;
* корректность подключения программатора или интерфейса загрузчика;
* отсутствие внешней нагрузки, которая может создать опасное состояние при перезапуске модуля;
* соответствие fuse-настроек микроконтроллера используемой частоте, размеру boot-секции и способу запуска прошивки.

Если используется прямой переход в bootloader из прикладной прошивки, необходимо учитывать, что в коде задан стартовый адрес bootloader:

```cpp
BOOT_START_ADDR_BYTES = 0x7000
```

Для ATmega328P это соответствует boot-секции размером **4096 байт** при верхней границе Flash `0x7FFF`. Значение должно соответствовать фактическим fuse-настройкам **BOOTSZ** и сборке используемого bootloader. Если fuse-настройки и адрес bootloader не совпадают, переход в bootloader по команде из Modbus может быть невозможен или приведёт к некорректному выполнению кода.

---

### Варианты загрузки прошивки

Прошивка может быть загружена одним из двух способов:

* через внешний программатор микроконтроллера;
* через bootloader, если он установлен и доступен.

Способ загрузки выбирается в зависимости от состояния микроконтроллера, наличия bootloader и требований производственного процесса.

---

### Загрузка через внешний программатор

Загрузка через внешний программатор используется как основной и наиболее надёжный способ первичной записи прошивки в микроконтроллер.

#### Порядок выполнения

1. Отключить питание модуля LDO.
2. Подключить программатор к разъёму программирования микроконтроллера.
3. Проверить правильность подключения линий питания, земли и интерфейса программирования.
4. Подать питание на модуль либо включить питание от программатора, если такой режим предусмотрен используемым оборудованием.
5. В программе загрузки выбрать целевой микроконтроллер **ATmega328P**.
6. Проверить доступность микроконтроллера чтением сигнатуры.
7. Выбрать файл прошивки:

```text
LDO_Rev3.hex
```

8. Выполнить запись Flash-памяти микроконтроллера.
9. При наличии функции верификации выполнить сравнение записанного содержимого с файлом `LDO_Rev3.hex`.
10. Отключить питание модуля.
11. Отключить программатор.
12. Повторно подать питание на модуль.
13. Выполнить проверку работы по Modbus RTU.

#### Контроль после записи

После записи через программатор необходимо убедиться, что:

* микроконтроллер успешно определяется программатором;
* запись Flash завершена без ошибок;
* верификация Flash завершена без ошибок;
* после перезапуска модуль не находится в состоянии постоянного сброса;
* устройство отвечает по Modbus RTU на адресе, заданном аппаратным переключателем.

---

### Загрузка через bootloader

Загрузка через bootloader допускается только при наличии установленного и исправного bootloader, совместимого с аппаратной схемой модуля и текущими fuse-настройками микроконтроллера.

Прикладная прошивка содержит служебный механизм отложенного перехода в bootloader. Переход выполняется через Holding-регистр **HR 900**. После записи в этот регистр прошивка выдерживает задержку перед переходом, чтобы Modbus-ответ успел физически уйти в линию RS-485, затем запрещает прерывания, отключает периферию и передаёт управление в boot-секцию.

#### Условия применения

Перед использованием загрузки через bootloader необходимо проверить:

* bootloader записан в микроконтроллер;
* fuse-настройки соответствуют адресу bootloader `0x7000`;
* интерфейс загрузчика соответствует фактической аппаратной схеме модуля;
* линия RS-485 подключена корректно;
* ведущий компьютер может обмениваться с устройством по RS-485;
* текущая прикладная прошивка отвечает по Modbus RTU.

#### Порядок перехода в bootloader

1. Подключить модуль LDO к линии RS-485.
2. Подать питание на модуль.
3. Установить параметры обмена:

| Параметр          | Значение             |
| ----------------- | -------------------- |
| Протокол          | Modbus RTU           |
| Скорость          | 9600 бод             |
| Биты данных       | 8                    |
| Чётность          | Нет                  |
| Стоп-биты         | 1                    |
| Схема подключения | Двухпроводная RS-485 |

4. Установить в программе Modbus адрес устройства, соответствующий положению адресного переключателя.
5. Выполнить запись в Holding-регистр **HR 900**.
6. Дождаться ответа устройства на Modbus-команду.
7. После ответа устройство выполнит отложенный переход в bootloader.
8. Выполнить загрузку файла `LDO_Rev3.hex` штатным способом, предусмотренным используемым bootloader.
9. После завершения загрузки перезапустить модуль.
10. Выполнить проверку связи и релейных выходов.

---

### Служебная перезагрузка из Modbus

В прошивке предусмотрена программная перезагрузка через watchdog. Перезагрузка вызывается записью в Holding-регистр **HR 594**.

После записи в `HR 594` прикладной код устанавливает флаг перезагрузки. В задаче с периодом около **100 мс** прошивка включает короткий timeout watchdog, после чего микроконтроллер перезагружается.

Этот механизм используется для сервисного перезапуска модуля после изменения состояния системы или после загрузочных операций. Он не заменяет процедуру записи Flash-памяти.

### Проверка после прошивки

После загрузки прошивки необходимо выполнить первичную проверку работоспособности модуля.

#### Проверка запуска

1. Подать питание на модуль.
2. Убедиться, что модуль запускается без признаков циклического сброса.
3. Проверить светодиодную индикацию питания и состояния модуля.
4. При отсутствии обмена по RS-485 светодиод состояния связи может находиться в режиме индикации отсутствия связи.

#### Проверка параметров связи

Настроить ведущую программу Modbus RTU со следующими параметрами:

| Параметр                     | Значение                      |
| ---------------------------- | ----------------------------- |
| Интерфейс                    | RS-485                        |
| Протокол                     | Modbus RTU                    |
| Скорость                     | 9600 бод                      |
| Формат кадра                 | 8N1                           |
| Функция чтения               | 0x03 Read Holding Registers   |
| Функция одиночной записи     | 0x06 Write Single Register    |
| Функция множественной записи | 0x10 Write Multiple Registers |

Адрес устройства должен соответствовать аппаратному адресному переключателю. Если переключатель находится в нулевом положении, используется адрес **1**.

#### Проверка ответа по Modbus

Выполнить чтение Holding-регистра **HR 1**.

Ожидаемый результат:

* устройство отвечает без ошибки CRC;
* Modbus-ответ приходит с тем же slave-адресом;
* значение регистра соответствует текущему состоянию релейных выходов;
* после первого корректного обращения прошивка устанавливает состояние связи и включает штатную индикацию связи.

#### Проверка управления выходами

Для проверки релейных выходов используется Holding-регистр **HR 0**.

Запись в `HR 0` задаёт требуемое состояние релейных выходов в виде битовой маски. Чтение `HR 0` или `HR 1` возвращает текущее применённое состояние релейных выходов.

Для аппаратной ревизии `BOARD_NEW_RS485_PD2` соответствие битов управляющей маски линиям микроконтроллера следующее:

| Бит маски | Линия ATmega328P |
| --------- | ---------------- |
| bit 0     | PB2              |
| bit 1     | PB1              |
| bit 2     | PB0              |
| bit 3     | PD7              |
| bit 4     | PD6              |
| bit 5     | PD5              |
| bit 6     | PD4              |
| bit 7     | PD3              |

Порядок проверки:

1. Записать в `HR 0` значение `0x0001`.
2. Проверить срабатывание соответствующего релейного выхода.
3. Прочитать `HR 1`.
4. Убедиться, что прочитанное значение соответствует записанной маске.
5. Повторить проверку для масок `0x0002`, `0x0004`, `0x0008`, `0x0010`, `0x0020`, `0x0040`, `0x0080`.
6. Записать в `HR 0` значение `0x0000`.
7. Убедиться, что все релейные выходы выключены.

Фактическое соответствие битов клеммам выходов **1...8** должно проверяться по электрической схеме платы или по контрольному стенду. В прошивке зафиксировано соответствие битов портам микроконтроллера.

#### Проверка таймаута связи

В прошивке реализован контроль потери связи с ведущим устройством. Если в течение **10000 мс** не было корректного обращения по Modbus RTU, прошивка сбрасывает требуемое состояние релейных выходов в ноль.

Порядок проверки:

1. Включить один или несколько релейных выходов записью в `HR 0`.
2. Прекратить обмен по Modbus RTU.
3. Выдержать паузу более **10000 мс**.
4. Проверить состояние релейных выходов.

Ожидаемый результат:

* требуемое состояние выходов сбрасывается в `0x00`;
* физически применённое состояние релейных выходов переходит в выключенное состояние при ближайшем выполнении периодической задачи;
* индикация состояния связи переходит в режим отсутствия связи.

### Карта регистров, используемая при проверке

Для загрузочных и проверочных операций используются следующие Holding-регистры:

| Регистр | Доступ | Назначение                                                          |
| ------- | ------ | ------------------------------------------------------------------- |
| HR 0    | R/W    | Уставка / чтение текущего состояния релейных выходов, битовая маска |
| HR 1    | R      | Чтение текущего состояния релейных выходов, битовая маска           |
| HR 3    | R      | Код ошибки устройства                                               |
| HR 580  | R/W    | Вход в сервисный режим при записи сервисного ключа                  |
| HR 590  | R      | Резерв сервисных данных                                             |
| HR 591  | R      | Резерв сервисных данных                                             |
| HR 592  | R      | Резерв сервисных данных                                             |
| HR 593  | R      | Резерв сервисных данных                                             |
| HR 594  | CMD    | Программная перезагрузка через watchdog                             |
| HR 900  | CMD    | Отложенный переход в bootloader                                     |

Поддерживаемые функции Modbus RTU:

| Код функции | Назначение               |
| ----------- | ------------------------ |
| 0x03        | Read Holding Registers   |
| 0x06        | Write Single Register    |
| 0x10        | Write Multiple Registers |

Максимальное количество регистров за один запрос чтения или множественной записи — **15**. Адресное пространство транспорта ограничено диапазоном Holding-регистров **0...1000**. Регистры, не поддержанные прикладной картой, возвращают значение `-1`.

| Реле отключаются через некоторое время         | Сработал таймаут потери связи                                                                                    | Поддерживать циклический обмен по Modbus RTU с периодом меньше 10000 мс                |
| Устройство постоянно перезагружается           | Ошибка питания, некорректная работа watchdog, повреждённая прошивка                                              | Проверить питание, повторно записать `LDO_Rev3.hex`, проверить сборку и fuse-настройки |

### Требования к безопасной загрузке

Перед прошивкой и проверкой необходимо учитывать, что LDO1128 управляет внешними цепями через релейные выходы с нормально открытыми и нормально закрытыми контактами. На контактах реле может присутствовать внешнее напряжение нагрузки.

Перед подключением программатора, RS-485 или измерительных приборов необходимо:

* отключить опасные внешние цепи нагрузки;
* исключить самопроизвольное включение исполнительных механизмов;
* проверить общую землю интерфейса RS-485;
* не выполнять загрузку прошивки при нестабильном питании;
* после прошивки сначала проверять связь и регистры, затем подключать нагрузку;
* при проверке выходов использовать безопасное контрольное напряжение или стендовую нагрузку.

---

### Критерии успешной загрузки

Загрузка прошивки считается выполненной успешно, если выполняются следующие условия:

* файл `LDO_Rev3.hex` записан в Flash-память микроконтроллера без ошибок;
* верификация Flash, если она выполнялась, завершена успешно;
* после перезапуска модуль запускается без циклического сброса;
* устройство отвечает по Modbus RTU на адресе, заданном аппаратным переключателем;
* чтение `HR 1` выполняется без ошибки;
* запись битовой маски в `HR 0` приводит к переключению соответствующих релейных выходов;
* чтение `HR 1` возвращает фактически применённое состояние выходов;
* при прекращении обмена более чем на **10000 мс** выходы переходят в безопасное выключенное состояние;
* команда программной перезагрузки через `HR 594` приводит к перезапуску устройства;
* команда перехода в bootloader через `HR 900` выполняется корректно при наличии установленного совместимого bootloader.
