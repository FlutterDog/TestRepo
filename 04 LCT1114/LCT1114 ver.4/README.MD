# Сравнение LCT1114 Revision 2 / Revision 3

## 1. Логика ревизий

| Ревизия        | Назначение                                                                                                                                                                                                                |
| -------------- | ------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **Версия 2** | Текущая версия прибора. Используется как сдаваемая базовая версия. Обеспечивает учет механического и электрического ресурса, ограниченную регистрацию осциллограмм и работу в составе системы через внешний модуль LCP.   |
| **Версия 3** | Новая компактная ARM-платформа в форм-факторе модуля Lorentz. Цель — устранить ограничения Rev.2 по памяти, автономности, разрешению измерений, шумам, ЭМС, осциллографированию соленоидов и локальному хранению событий, разрешению АЦП. режимам номинального тока и тока КЗ, улучшенным настройкам и калибровке аналоговых входов с помощью 8 канального АЦП одновременных измерений. |

## 7. Основное обоснование перехода с Rev.2 на Rev.3

| Ограничение Revision 2                | Практическое последствие                                                       | Решение в Revision 3                                                 |
| ------------------------------------- | ------------------------------------------------------------------------------ | -------------------------------------------------------------------- |
| Ограниченный объем оперативной памяти | Нельзя хранить достаточный объем осциллограмм и выполнять расширенный анализ   |  SRAM около 1 MB                                            |
| FRAM 8 КБ                             | Хранение только нескольких осциллограмм                                        | Event Recorder с RAM-буферами и Flash-хранилищем 128-256 последних коммутаций в существенном разрешении |
| 12-битный АЦП                         | Недостаточное разрешение и качество осциллограмм                               | 16-битный внешний ADC для главных токов                              |
| Нет High/Low gain                     | Нельзя одновременно получить широкий диапазон и хорошее разрешение малых токов | Два параллельных диапазона Low/High для трех фаз                     |
| Соленоиды без осциллограмм            | Невозможен анализ формы тока электромагнитов                                   | Осциллографирование соленоидов                                       |
| RTC без батареи                       | Модуль не является полноценным автономным регистратором                        | Встроенные часы реального времени с автономным питанием              |
| Зависимость от LCP                    | LCT не может самостоятельно хранить и обслуживать события                      | Автономная запись, хранение и выгрузка событий                       |
| SD-карта не задействована полноценно  | Нестабильное/неполное хранение данных                                          | Встроенная промышленная Flash-память на плате                        |
| Ethernet не доведен                   | Сетевые функции ограничены ресурсами текущей платформы                         | достаточная память для Modbus TCP / 104 / сервисной выгрузки       |
| Ограниченная ЭМИ-защита / шумы        | Плохое качество малых сигналов и ограниченная помехоустойчивость               | Переработанный аналоговый фронтенд, улучшенная ЭМИ-защита питания и разводка |




## Предложение по разработке образца LCT3 с указанными характеристиками

Настоящее предложение описывает ориентировочный состав работ по разработке инженерного образца **LCT3** на новой аппаратной платформе.

Разработка выполняется на базе существующих наработок по модулю **LCT1114 Revision 2**. В текущей версии уже реализованы основные алгоритмы учета механического и электрического ресурса, обработка концевиков, расчет временных параметров, работа с осциллограммами фазных токов, расчетные значения токов электромагнитов и обмен данными по существующей карте регистров.

Цель работ — подготовить новую аппаратную платформу на базе **ARM Cortex-M7**, перенести существующую логику на новую платформу и подготовить комплект данных для изготовления и первичной проверки опытного образца.

### Основные задачи этапа

* разработка и проверка аппаратной архитектуры LCT3;
* переход на ARM Cortex-M7;
* увеличение объема оперативной памяти для регистрации событий;
* реализация локальной работы с буферами осциллограмм;
* подготовка схемы новой аппаратной платформы;
* подготовка трассировки платы опытного образца;
* перенос базовой логики существующей прошивки;
* адаптация работы с измерительными каналами;
* подготовка базовой структуры хранения событий;
* сохранение совместимости с существующей логикой обмена;
* подготовка производственных файлов для изготовления опытных плат;
* изготовление или получение опытного образца для отладки;
* первичная проверка и запуск опытного образца;
* доведение Toolbox / сервисного ПО под работу с LCT3.

## Изготовление и отладка опытных плат

После подготовки производственного комплекта изготавливается опытная партия плат для проверки аппаратной платформы и первичной отладки прошивки.

Для выполнения отладки разработчику должен быть доступен как минимум один собранный опытный образец LCT3.

Возможны два варианта обеспечения опытного образца для отладки:

| Вариант   | Описание                                                                                                                                                                                                             |
| --------- | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| Вариант 1 | Разработчик самостоятельно организует изготовление и сборку одного опытного образца. Стоимость изготовления, сборки, компонентов и доставки включается в общую стоимость договора или согласуется отдельной строкой. |
| Вариант 2 | Заказчик самостоятельно изготавливает опытную партию плат и передает один собранный образец разработчику для отладки, проверки исправлений и сопровождения прошивки.                                                 |

Минимально целесообразно изготовить **2 опытные платы**:

| Плата    | Назначение                                                                    |
| -------- | ----------------------------------------------------------------------------- |
| Плата №1 | Проверка и испытания на стороне заказчика                                     |
| Плата №2 | Отладка, проверка исправлений и техническая поддержка на стороне разработчика |

Наличие опытного образца у разработчика необходимо для проверки питания, запуска процессора, проверки интерфейсов, измерительных каналов, записи событий, обмена данными и последующего сопровождения прошивки.

Расходы на изготовление, сборку, компоненты, доставку и возможные таможенные платежи должны быть согласованы до заказа опытных плат.

## Первичная проверка опытного образца

Первичная проверка опытного образца выполняется после получения собранной платы и включает следующие работы:

|  № | Проверка                       | Содержание                                                                    |
| -: | ------------------------------ | ----------------------------------------------------------------------------- |
|  1 | Проверка питания               | Контроль входного питания, внутренних напряжений, потребления и нагрева       |
|  2 | Запуск процессора              | Проверка программирования, отладки, clock, reset и базовой конфигурации       |
|  3 | Проверка памяти                | Проверка доступа к энергонезависимой памяти и базовой структуры хранения      |
|  4 | Проверка интерфейсов           | Проверка RS-485 / X2X, Ethernet, USB в объеме аппаратной реализации           |

Предлагаемый этап позволяет перейти от текущего модуля **LCT1114 Revision 2** к инженерному образцу **LCT3** на базе **ARM Cortex-M7**.

Основной результат этапа — комплект аппаратной документации, производственные файлы, базовый проект прошивки, перенесенная основная логика существующего модуля, адаптированный Toolbox и опытный образец, доступный для первичной отладки и проверки.
