# Stage 5 — QC + Release Build

## Purpose

Stage five выполняет финальную проверку качества и собирает релиз.

Но эта стадия **не должна впервые придумывать глубину текста**. Её задача — проверить, что Stage four уже дал содержательный, объясняющий draft, и довести его до clean release.

---

## QC focus

Финальная проверка должна включать не только factual cleanliness, но и explanatory adequacy.

### Обязательно проверить

* текст не сухой и не сведён к инвентарным карточкам;
* в каждой крупной главе есть назначение, реализация, характеристики, возможности и ограничения;
* нет неподтверждённых обещаний;
* нет process-language;
* figures и ссылки согласованы;
* релиз объясняет плату как платформу и как конкретную аппаратную реализацию.
