# GLOSSARY

## Назначение документа

ℹ️ Project-specific словарь для Stage two — Stage five.

⚠️ Включены только термины, правила записи и naming rules, реально нужные для текущего freeze-комплекта.

❗ При конфликте использовать иерархию SoT из этого файла, а не «память по проекту».

---

## Статусы утверждений

### FACT

ℹ️ Подтверждено источником или закрыто operator review с явным resolution note.

Требование:

* указывать файл и точный якорь: page, sheet title, designator, net name, archive entry или report token;
* если использовано operator clarification, явно отмечать это в notes.

### ASSUMPTION

⚠️ Рабочее допущение.

Требование:

* указывать риск;
* указывать, что нужно для подтверждения;
* заносить в `open_items.md`.

### UNKNOWN

❗ Не подтверждено или конфликтует между источниками.

Требование:

* не превращать в release-утверждение;
* заносить в `open_items.md`.

## Иерархия Source of Truth

### Порядок приоритета

* `sources/cad/Native.zip` — первично для электрической логики, состава листов, RefDes и net names
* `sources/schematic/Schematic.pdf` — первично для читабельного подтверждения листов, попавших в PDF-export
* `sources/pcb/Job1.PDF` — первично для PCB artwork, маркировки на плате и физического расположения интерфейсов
* `sources/stackup/LCP.stackup` — первично для layer stack, меди, диэлектриков и структуры платы
* `sources/gerber/Gerber.zip` — первично для fabrication-фактов, drill и layer-output
* `sources/bom/BOM.csv` — первично для состава компонентов, footprints, part numbers и quantity
* `sources/extras/*.pdf` — только reference/proof-слой верхнего уровня; не переопределяет native CAD без явного основания
* operator clarification — допустим только для закрытия конфликтов naming / release wording; он не должен подменять электрическую схему или fabrication-факт

### Специальное правило для текущего проекта

* Каноническое имя текущей платы — `LCP2216`.
* Каноническая ревизия текущей платы — `Rev.7.1`.
* Токен `LCP2116` в reference-manual трактовать как аналогичный модуль другой формы исполнения, а не как имя текущей платы.
* Токен `rev 7.2` внутри export-path трактовать как артефакт пути, а не как действующую ревизию текущей платы.

## Предпочтительная терминология проекта

### Базовые наименования

* использовать: **процессорный модуль**
* не использовать по умолчанию как основной термин: `контроллер`, если источник прямо не требует этого слова

### По питанию

* использовать: **линия питания**
* не использовать как основной термин без необходимости: `домен питания`

### По интерфейсам

* использовать: `линия RS-485`, `Ethernet-порт`, `сервисный USB`, `линия X2X`
* использовать: `дифференциальная линия`, если речь именно о паре `P/N` или `A/B`
* использовать: `назначение порта гибкое`, если речь идёт о двух Ethernet-портах

### По физической реализации

* использовать: `PCB artwork`, `assembly view`, `stackup`, `fabrication archive`, `schematic PDF`, `native CAD`
* использовать: `as-built`, когда речь идёт о фактической реализации платы по freeze-комплекту

## Board-specific naming rules

### Идентичность изделия

* board name для текущего guide: `LCP2216`
* board revision для текущего guide: `Rev.7.1`
* полный операторский идентификатор пайплайна: `LCP_rev7.1`

### Reference-manuals

* `LCP2116` в `sources/extras/Manual_Lorentz.pdf` не нормализовать молча в `LCP2216`
* при необходимости писать так: `reference-manual по аналогичному модулю семейства Lorentz`

### Page `9` внутри `Schematic.pdf`

* трактовать как `assembly view` / `PCB artwork proof`
* использовать для маркировки платы, разъёмов и физического расположения элементов
* не трактовать как отдельный schematic-sheet

## Правила записи единиц

### Основные единицы

* напряжение: `V`
* ток: `mA`, `A`
* частота: `kHz`, `MHz`
* ёмкость: `pF`, `nF`, `uF`
* сопротивление: `R`, `k`, `M`
* геометрия PCB: `mm`
* медь: `mm` по freeze-источнику; не пересчитывать в `oz`, если источник этого не даёт

### Правила оформления

* сохранять запись так, как она дана в freeze-источнике, если это важно для поиска:
  * `+5V`
  * `+3V3`
  * `25.0 MHz`
  * `32.768 kHz`
* не нормализовать автоматически `uSD` в `microSD`, если цитируется точный текст источника; в prose допускается `microSD`

## Правила именования сигналов и линий

### Питание

* `+5V`
* `+3V3`
* `VBAT` и связанные сигналы сохранять как в источнике
* при описании в тексте писать: `линия питания +5V`, `линия питания +3V3`

### Ethernet

* пары:
  * `TX_P / TX_N`
  * `RX_P / RX_N`
  * `TX_2_P / TX_2_N`
  * `RX_2_P / RX_2_N`
* chip-select lines:
  * `CS_ETH`
  * `CS_ETH_2`
* два Ethernet-порта аппаратно одинаковы; функциональное назначение задаётся системой, а не разводкой листов `Ethernet` / `ETH_HMI`

### RS-485 и X2X

* аппаратно видимые линии в schematic PDF:
  * `RS1_P / RS1_N` … `RS6_P / RS6_N`
  * `X_L / X_H`
* release-терминология для текущего guide:
  * `RS1`…`RS4` — линии для внешних устройств
  * `HMI` — выделенная линия на физическом уровне `RS-485`
  * `PC` — выделенная линия на физическом уровне `RS-485`
  * `X2X` — линия подключения модулей
* внутренние логические сигналы оставлять как в схеме:
  * `UART_RTS_2`
  * `UART_RTS_3`
  * `RTS_PC`
  * `RTSX`

### SD

* `MISO`
* `MOSI`
* `SPCK`
* `CS_SD`
* `SD_Detect`

## Правила ссылок «где смотреть»

### Формат якоря

Использовать один из вариантов:

* `page N, title <sheet title>`
* `designator <RefDes>`
* `archive entry <file inside zip>`
* `report token <exact token>`
* `layer token <exact layer token>`
* `operator clarification <short resolution>`

### Примеры для текущего проекта

* `sources/schematic/Schematic.pdf` → `page 5, title PSU 24V-5V`
* `sources/schematic/Schematic.pdf` → `page 6, title CPU ARM_M3`

## Термины, реально используемые в проекте

### Вычислитель

* **процессорный модуль** — основной модуль с `MCU`, памятью, интерфейсами и служебными узлами
* **микроконтроллер** — `U1 = ATSAM3X8EA-AU`

### Питание

* **линия питания** — электрическая линия распределения питания, например `+5V`, `+3V3`
* **понижающий преобразователь** — `U2 TPS54331DR`
* **LDO** — `U3 LM1117MPX-3.3`
* **резервирование RTC** — узел батареи `CR2032` и контроля напряжения батареи

### Интерфейсы

* **Ethernet-порт** — физический RJ45-порт с отдельным `W5500`
* **линия RS-485** — одна линия на физическом уровне `RS-485`
* **линия X2X** — линия подключения модулей на физическом уровне `RS-485`
* **сервисный USB** — USB B для firmware / debug / service функций
* **uSD / microSD** — съёмный носитель на SPI

### PCB и fabrication

* **assembly view** — графический вид платы с маркировкой и позиционированием компонентов
* **stackup** — структура слоёв платы
* **plane layer** — внутренний сплошной слой, здесь `Gnd` или `Power`
* **fabrication archive** — Gerber + drill + report output
* **PCB artwork** — графический вид платы, шелкография, placement и трассировка

## Термины, которых стоит избегать без подтверждения

* `гальваническая изоляция интерфейсов` — не использовать без отдельного proof-source
* точный численный диапазон входной линии питания
* `LCP2116` как имя текущей платы
* `rev 7.2` как ревизия текущей платы
