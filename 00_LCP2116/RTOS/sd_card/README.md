# Пример содержимого microSD

Скопируйте штатные TXT-файлы из этого каталога в корень microSD контроллера. Имена файлов являются частью существующего формата проектов и не имеют псевдонимов.

## X2X

`X2X.TXT` задаёт состав системных модулей X2X.

## FieldSensor S1–S4

```text
baud.TXT
Parity.TXT
```

Порядок значений: S1, S2, S3, S4.

`baud.TXT` содержит фактические значения скорости, например `1200` или `9600`.

Коды `Parity.TXT`:

```text
0 = None, 8N1
1 = Odd,  8O1
2 = Even, 8E1
```

## EIA.TXT

`EIA.TXT` хранит привязку логических потоков `ch1..ch4` к физическим портам S1..S4:

```text
3 = S1
4 = S2
5 = S3
6 = S4
```

Базовый пример Lorentz не использует `EIA.TXT`, поскольку в нём нет логических потоков и каждый `FieldSensor` напрямую закреплён за S1, S2, S3 или S4. Файл сохранён как штатный формат для прикладных проектов, включая PNX.

## Ethernet

```text
ETH1: MAC.txt,  IP.txt,  SUBNET.txt,  GATE.txt
ETH2: MAC2.txt, IP2.txt, SUBNET2.txt, GATE2.txt
```

Альтернативные имена файлов не поддерживаются.

IP, SUBNET и GATE содержат четыре октета. MAC содержит шесть байтов, записанных десятичными числами `0..255`.

Все файлы используют формат:

```text
количество значений
значение 1
значение 2
...
fin
```

Строка `fin` завершает старый формат и игнорируется после чтения указанного количества значений.

Присланный пример настроен на:

```text
S1: 9600 8N1
S2: 9600 8N1
S3: 1200 8N1
S4: 9600 8N1

ETH1 IP: 192.168.1.1
ETH2 IP: 192.168.1.1
Subnet: 255.255.255.0
Gateway: 0.0.0.0
Modbus TCP port: 502
```

Одинаковые IP допустимы только при работе ETH1 и ETH2 в раздельных физических сетях. При подключении обоих портов к одной сети адреса должны различаться.
