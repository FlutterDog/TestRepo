# Проверка базовой подсистемы X2X

## Статус

Реализация находится в `feature/freertos-baseline`.

Аппаратно подтверждены:

- пустая конфигурация X2X;
- LDI1116;
- потеря и автоматическое восстановление связи;
- LAI1118 и порядок слов `float`.

После корректировки карт LCT требуется повторно собрать Debug и Release, затем проверить обе версии LCT на оборудовании.

Подробные зафиксированные результаты находятся в `X2XHardwareResults.md`.

## 1. Предварительные условия

- контроллер LCP2116 с ATSAM3X8E;
- microSD с FAT16/FAT32;
- USB CDC подключён к терминалу 115200 8N1;
- X2X работает на UART0, 9600 8N1;
- модули имеют последовательные адреса 1..N в порядке `X2X.TXT`;
- максимальное количество внешних модулей — 6.

## 2. Стабильные ID модулей

```text
0 — LCP2116, в X2X.TXT запрещён
1 — LDO1118
2 — LAI1118
3 — LDI1118
4 — LCT1114, первая версия
5 — LDI1116
6 — LCT1114_2, вторая версия
```

Назначенные ID являются частью формата конфигурации и не должны изменяться.

## 3. Проверка сборки

Собрать:

```text
Debug
Release
```

Проверить:

- отсутствуют ошибки компиляции и линковки;
- нет undefined reference для X2X и Modbus RTU;
- RAM не превышает 96 KiB;
- Flash не превышает 512 KiB;
- увеличение `X2X_MAX_WAVEFORM_SAMPLES` с 200 до 400 не приводит к переполнению RAM;
- отсутствуют duplicate symbol для UART handlers, SysTick, PendSV и SVC.

Зафиксировать:

```text
text=
data=
bss=
```

## 4. Проверка X2X.TXT

После изменения файла:

```text
x2x reload
x2x
```

### 4.1 Пустая конфигурация

```text
0
Fin
```

Ожидается:

```text
loaded=yes
result=ok
registry=ok
modules=0
port_owner=echo_test
```

### 4.2 Корректная конфигурация

```text
4
1
2
5
6
Fin
```

Ожидается `modules=4`, а slave-адреса должны соответствовать порядку строк.

### 4.3 Ошибочные файлы

Проверить:

- отсутствующий файл;
- ID 0;
- неизвестный ID;
- более 6 модулей;
- обрезанный файл;
- лишние данные;
- слишком длинную строку;
- некорректное числовое значение;
- повторный или лишний `Fin`.

При ошибке нового файла ранее активная корректная конфигурация должна продолжить работать.

## 5. Владение UART0

При `modules > 0`:

```text
X2X: mode=Modbus RTU master, echo=disabled
port_owner=x2x_master
```

При `modules=0`:

```text
X2X: mode=echo test, echo=enabled
port_owner=echo_test
```

S2, S3 и S4 должны продолжать echo-test независимо от X2X.

## 6. LDI1118 и LDI1116

Проверить каждый модуль отдельно:

```text
LDI1118: ID 3, биты 0..7
LDI1116: ID 5, биты 0..15
```

Требуется:

- `connection=online`;
- рост `success`;
- `consecutive_failures=0`;
- изменение `digital_inputs` для каждого физического входа.

## 7. LAI1118

Конфигурация:

```text
1
2
Fin
```

Проверить:

- чтение 17 holding-регистров с адреса 0;
- DI из регистра 0;
- восемь `float` из регистров 1..16;
- порядок слов: младшее слово первым;
- отсутствие `NaN` и бессмысленных значений;
- сравнение минимум двух каналов с известными значениями.

## 8. LDO1118

Конфигурация:

```text
1
1
Fin
```

Команды:

```text
x2x ldo 1 0
x2x ldo 1 1
x2x ldo 1 2
x2x ldo 1 4
x2x ldo 1 255
x2x ldo 1 0
```

Проверить:

- функцию 0x10;
- соответствие физических выходов битовой маске;
- подтверждение адреса и количества регистров;
- сохранение последней заданной команды при потере связи;
- отсутствие утверждения о фактическом состоянии выхода, пока связь не восстановлена.

## 9. LCT1114 — первая версия

Источник карты: `00_LCP2116/LCT1_r1_b/main.cpp`, функция `modbusProceed`.

### 9.1 Основной блок

```text
start=0
count=86
range=0..85
```

Разметка:

```text
0..1   два слова SP/DI
2..85  42 float
```

Ожидается:

```text
SP/ME/TF=20/0/42
```

### 9.2 Осциллограммы

```text
A: 200..395 — 196 точек
B: 400..595 — 196 точек
C: 600..795 — 196 точек
```

Каждая фаза читается блоками 100 + 96.

После полного чтения:

```text
waveform_valid=yes
samples_per_phase=196
sequence увеличивается
```

Затем LCP должен записать 0 в регистр 850.

## 10. LCT1114_2 — вторая версия

Источник карты: `00_LCP2116/LCT2_r8_p330/main.cpp`, функция `modbusProceed`.

### 10.1 Основной блок

```text
start=0
count=94
range=0..93
```

Разметка:

```text
0..1   два слова SP/DI
2..91  45 float
92     contacts_state_raw
93     previous_contacts_state
```

Регистры 92 и 93 не должны декодироваться как 46-й `float`.

Ожидается:

```text
SP/ME/TF=20/0/45
```

### 10.2 Осциллограммы

Активные диапазоны текущей прошивки:

```text
A: 3000..3399 — 400 точек
B: 3400..3799 — 400 точек
C: 3800..4199 — 400 точек
```

Каждая фаза читается четырьмя блоками по 100 регистров.

После полного чтения:

```text
waveform_valid=yes
samples_per_phase=400
sequence увеличивается
```

Затем LCP должен записать 0 в регистр 850. После захвата сохраняется пауза 1000 мс перед следующей проверкой флага.

Полная расшифровка находится в `LCTRegisterMaps.md`.

## 11. Ошибки и восстановление

Для каждого типа модуля:

1. Дождаться `connection=online`.
2. Разорвать линию.
3. Проверить рост `failed` и `consecutive_failures`.
4. После пяти ошибок получить `connection=lost`.
5. Восстановить линию.
6. После первого корректного ответа получить:

```text
connection=online
consecutive_failures=0
communication_error=ok
```

Последние корректные данные должны сохраняться без обнуления, но считаться невалидными до восстановления связи.

Отдельно проверить:

- неправильный CRC;
- ответ другого slave;
- неправильный function code;
- Modbus exception;
- неполный ответ;
- отсутствие ответа.

## 12. Runtime-команды

Проверить на активной линии:

```text
x2x pause
x2x resume
x2x reload
```

Pause и reload не должны обрывать активный Modbus-кадр. Допустимы ответы `pause pending` и `apply pending`; операция должна применяться на безопасной границе полного цикла модуля.

## 13. Несколько модулей

Проверить цепочку минимум из LDI, LAI и LCT:

- последовательный опрос всех slave;
- рост `success` каждого модуля;
- потерю только одного модуля без остановки остальных;
- восстановление отключённого модуля;
- отсутствие UART errors;
- корректное владение общим буфером осциллограмм.

## 14. Длительная работа

Минимум 30 минут:

- USB CDC остаётся доступным;
- watchdog не перезапускает контроллер;
- heap и минимальный стек не уменьшаются;
- счётчики успешных опросов растут;
- нет неконтролируемого роста ошибок;
- после чтения LCT2 буфер на 400 точек не повреждает соседние данные RAM.
