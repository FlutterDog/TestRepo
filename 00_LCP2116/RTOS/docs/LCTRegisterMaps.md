# Карты регистров LCT для X2X

## Источник

Карты восстановлены непосредственно из функций `modbusProceed` и библиотек `IGAS_485` в прошивках модулей:

- `00_LCP2116/LCT1_r1_b/main.cpp` — первая версия LCT;
- `00_LCP2116/LCT1_r1_b/IGAS_485.h`;
- `00_LCP2116/LCT2_r8_p330/main.cpp` — вторая версия LCT;
- `00_LCP2116/LCT2_r8_p330/IGAS_485.h`;
- `00_LCP2116/LCT2_r8_p330/osc_config.h`.

Этот документ является источником для драйвера:

```text
00_LCP2116/RTOS/app/x2x/modules/x2x_lct.cpp
```

## Ограничение Modbus slave

Обе версии LCT используют библиотеку `IGAS_485` со следующими пределами:

```text
MAX_READ_REGS = 16
MAX_WRITE_REGS = 16
MAX_MESSAGE_LENGTH = 40 bytes
```

Нормальный ответ функции 0x03 на 16 регистров занимает:

```text
1 slave + 1 function + 1 byte count + 32 data + 2 CRC = 37 bytes
```

Поэтому LCP не должен запрашивать у LCT более 16 регистров одним кадром, даже если собственный Modbus master LCP допускает чтение 100 регистров.

Ограничение применяется к:

- основному блоку данных;
- флагу готовности;
- каждой фазе осциллограммы.

Полный блок сначала собирается в буфере LCP и декодируется только после успешного получения всех частей. При ошибке отдельного чанка неполный блок не публикуется.

## Общие правила

- протокол: Modbus RTU;
- функция 0x03 используется для чтения;
- функция 0x10 используется для сброса флага осциллограммы;
- порядок слов `float`: младшее 16-битное слово первым, старшее вторым;
- регистр 0 содержит состояние контактов;
- регистр 1 содержит `complexSP`;
- флаг новых данных осциллограммы находится в регистре 850;
- значение 1 в регистре 850 означает наличие новых данных;
- после успешного чтения трёх фаз LCP записывает 0 в регистр 850;
- при ошибке обмена последние корректные значения сохраняются, но считаются невалидными по качеству связи.

## LCT1114 — первая версия

Источник: `LCT1_r1_b`.

### Основной блок

```text
start = 0
count = 86
range = 0..85
```

Разметка:

```text
0       curr_state
1       complexSP
2..85   42 float, по два регистра на значение
```

Основной блок читается шестью запросами:

```text
0..15    count 16
16..31   count 16
32..47   count 16
48..63   count 16
64..79   count 16
80..85   count 6
```

### Осциллограммы

```text
фаза A: 200..395   — 196 регистров
фаза B: 400..595   — 196 регистров
фаза C: 600..795   — 196 регистров
```

Каждая фаза читается тринадцатью запросами:

```text
12 × 16 + 4 = 196 регистров
```

## LCT1114_2 — вторая версия

Источник: `LCT2_r8_p330`.

### Основной блок

```text
start = 0
count = 94
range = 0..93
```

Разметка:

```text
0       curr_state
1       complexSP
2..91   45 float, по два регистра на значение
92      contacts_state_raw, uint16
93      prev_state, uint16
```

Регистры 92 и 93 не образуют 46-е значение `float`.

Основной блок читается шестью запросами:

```text
0..15    count 16
16..31   count 16
32..47   count 16
48..63   count 16
64..79   count 16
80..93   count 14
```

Первые 42 значения соответствуют первой версии. Дополнительные значения:

```text
43  ток соленоида включения
44  ток первого соленоида отключения
45  ток второго соленоида отключения
```

Runtime-поля для регистров 92 и 93:

```text
contacts_state_raw
previous_contacts_state
```

### Осциллограммы

В текущей прошивке активны диапазоны `dbg_tap_get`:

```text
фаза A: 3000..3399   — 400 регистров
фаза B: 3400..3799   — 400 регистров
фаза C: 3800..4199   — 400 регистров
```

`PHASE_SAMPLES_MAX` в `osc_config.h` равен 400.

Каждая фаза читается двадцатью пятью запросами:

```text
25 × 16 = 400 регистров
```

Старые диапазоны `200..395`, `400..595`, `600..795` в прошивке LCT2 закомментированы и драйвером LCP не используются.

## Проверка по трассировке

При обычном опросе без готовой осциллограммы должны повторяться:

```text
01 03 00 00 00 10 ...   чтение регистров 0..15
01 03 00 10 00 10 ...   чтение регистров 16..31
01 03 00 20 00 10 ...   чтение регистров 32..47
01 03 00 30 00 10 ...   чтение регистров 48..63
01 03 00 40 00 10 ...   чтение регистров 64..79
01 03 00 50 00 0E ...   чтение регистров 80..93
01 03 03 52 00 01 ...   чтение регистра 850
```

Ответ регистра 850:

```text
01 03 02 00 00 ...      новых данных нет
01 03 02 00 01 ...      осциллограмма готова
```

Только после ответа `00 01` должны появиться запросы диапазонов 3000, 3400 и 3800.

Первый запрос осциллограммы LCT2:

```text
01 03 0B B8 00 10 ...   start 3000, count 16
```

Последний запрос фазы A:

```text
01 03 0D 38 00 10 ...   start 3384, count 16
```

## Проверка на оборудовании

Для первой версии ожидается:

```text
SP/ME/TF=20/0/42
samples_per_phase=196
```

Для второй версии ожидается:

```text
SP/ME/TF=20/0/45
samples_per_phase=400
```

После установки флага 850 в 1 необходимо проверить:

1. чтение всех трёх фаз без Modbus-ошибок;
2. увеличение `waveform.sequence`;
3. `waveform_valid=yes`;
4. запись нуля в регистр 850;
5. отсутствие выхода за границы общего буфера;
6. соответствие первых и последних отсчётов данным модуля.
