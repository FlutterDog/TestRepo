# Карты регистров LCT для X2X

## Источник

Карты восстановлены непосредственно из функций `modbusProceed` в прошивках модулей:

- `00_LCP2116/LCT1_r1_b/main.cpp` — первая версия LCT;
- `00_LCP2116/LCT2_r8_p330/main.cpp` — вторая версия LCT.

Этот документ является источником для драйвера:

```text
00_LCP2116/RTOS/app/x2x/modules/x2x_lct.cpp
```

## Общие правила

- протокол: Modbus RTU, функция 0x03 для чтения;
- порядок слов `float`: младшее 16-битное слово первым, старшее вторым;
- регистр 0 содержит состояние контактов;
- регистр 1 содержит `complexSP`;
- флаг новых данных осциллограммы находится в регистре 850;
- значение 1 в регистре 850 означает наличие новых данных;
- после успешного чтения трёх фаз LCP записывает 0 в регистр 850;
- при ошибке обмена последние корректные значения сохраняются, но считаются невалидными по качеству связи.

### Осциллограммы

```text
фаза A: 200..395   — 196 регистров
фаза B: 400..595   — 196 регистров
фаза C: 600..795   — 196 регистров
```

Каждая фаза читается двумя запросами:

```text
100 + 96 регистров
```

## LCT1114_2 — вторая версия

Источник: `LCT2_r8_p330`.

### Основной блок

```text
start = 0
count = 94
range = 0..93
```

Разметка:

```text
0       curr_state
1       complexSP
2..91   45 float, по два регистра на значение
92      contacts_state_raw, uint16
93      prev_state, uint16
```

Регистры 92 и 93 не образуют 46-е значение `float`.

Первые 42 значения соответствуют первой версии. Дополнительные значения:

```text
43  ток соленоида включения
44  ток первого соленоида отключения
45  ток второго соленоида отключения
```

Runtime-поля для регистров 92 и 93:

```text
contacts_state_raw
previous_contacts_state
```

### Осциллограммы

В текущей прошивке активны диапазоны `dbg_tap_get`:

```text
фаза A: 3000..3399   — 400 регистров
фаза B: 3400..3799   — 400 регистров
фаза C: 3800..4199   — 400 регистров
```

`PHASE_SAMPLES_MAX` в `osc_config.h` равен 400.

Каждая фаза читается четырьмя запросами по 100 регистров.

Старые диапазоны `200..395`, `400..595`, `600..795` в прошивке LCT2 закомментированы и драйвером LCP не используются.

## Проверка на оборудовании

Для первой версии ожидается:

```text
SP/ME/TF=20/0/42
samples_per_phase=196
```

Для второй версии ожидается:

```text
SP/ME/TF=20/0/45
samples_per_phase=400
```

После установки флага 850 в 1 необходимо проверить:

1. чтение всех трёх фаз без Modbus-ошибок;
2. увеличение `waveform.sequence`;
3. `waveform_valid=yes`;
4. запись нуля в регистр 850;
5. отсутствие выхода за границы общего буфера;
6. соответствие первых и последних отсчётов данным модуля.
