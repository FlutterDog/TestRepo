# Границы базовой прошивки Lorentz

## Назначение

Базовая прошивка Lorentz является демонстрационной и эксплуатационной основой свободнопрограммируемого контроллера LCP2116.

Она показывает три независимых механизма:

1. работу модулей системы по внутренней шине X2X;
2. циклический опрос внешнего Modbus RTU-прибора через S1–S4;
3. публикацию данных через Modbus TCP на двух Ethernet-интерфейсах.

IEC 60870-5-104, PNX Studio, панель оператора и прикладная логика конкретной установки в эту прошивку не входят.

## 1. X2X

Поддерживаемые модули:

```text
LDO1118
LAI1118
LDI1118
LDI1116
LCT1114
LCT1114_2
```

Состав и порядок модулей задаются в `X2X.TXT`; позиция в файле определяет slave address.

Реализованы:

- неблокирующий poll/write;
- timeout, CRC, exception и проверка ответа;
- сохранение последнего корректного значения;
- отдельное состояние качества;
- автоматическое восстановление;
- команда `x2x`;
- статический registry и централизованный catalog драйверов.

## 2. FieldSensor на S1–S4

Демонстрационный прибор жёстко задан в исходном коде:

```text
slave address: 1
function:      0x03 Read Holding Registers
start:         0
count:         2
period:        300 ms
timeout:       500 ms
```

Один и тот же пример независимо работает на S1, S2, S3 и S4. Каждый порт имеет отдельный transport, `ModbusRtuMaster`, буферы, таймеры, значения, качество и счётчики.

После пяти последовательных ошибок устанавливаются `connection=lost` и `valid=no`. Последние значения не обнуляются. Первый корректный ответ восстанавливает связь.

### Аппаратные настройки

Из microSD читаются:

```text
BAUD.TXT
PARITY.TXT
```

Порядок значений: S1, S2, S3, S4. Значения по умолчанию — 9600 8N1. Тип прибора, адрес slave и карта регистров не переносятся в файл и остаются ясным примером в коде.

Команды:

```text
field
field reload
field pause
field resume
```

Минимальный reusable Modbus RTU slave FC03 также входит в исходники, но пример его подключения отключён по умолчанию.

Подробности: `docs/FieldSensorExample.md`.

## 3. Ethernet Modbus TCP

Оба W5500 работают как независимые Modbus TCP server:

```text
ETH1: TCP port 502
ETH2: TCP port 502
function: 0x03 Read Holding Registers
map: 0..11
```

Сетевые параметры читаются из совместимых TXT-файлов:

```text
ETH1: IP.TXT, SUBNET.TXT, GATE.TXT
ETH2: IP2.TXT, SUBNET2.TXT, GATE2.TXT
```

Также поддерживаются совместимые aliases `IP1`, `SUBNET1`, `GATE1`, `GW`, `GW1` и `GW2`.

Оба интерфейса предоставляют одинаковую карту:

```text
0   S1 register 0
1   S1 register 1
2   S1 quality
3   S2 register 0
4   S2 register 1
5   S2 quality
6   S3 register 0
7   S3 register 1
8   S3 quality
9   S4 register 0
10  S4 register 1
11  S4 quality
```

Карта read-only. Неподдерживаемые функции и адреса получают стандартные Modbus exception.

Команды:

```text
eth
eth reload
```

Подробности: `docs/ModbusTcpExample.md`.

## Что намеренно не входит

```text
IEC 60870-5-104
панель оператора
PNX Studio
проектные файлы PNX
база типов внешних приборов
динамическое добавление сенсоров пользователем
технологические алгоритмы конкретной установки
релейные матрицы PNX
газовые таблицы и специализированные драйверы PNX
```

## Оставшиеся действия перед релизом

1. Собрать Debug и Release Stage 18A.
2. Проверить загрузку BAUD/PARITY и фактические параметры S1–S4.
3. Проверить каждый FieldSensor-порт отдельно и совместно.
4. Проверить timeout, потерю и автоматическое восстановление.
5. Проверить ETH1 и ETH2 как Modbus TCP server на порту 502.
6. Сравнить данные TCP 0..11 с командой `field`.
7. Проверить совместную работу X2X, FieldSensor и Ethernet.
8. Зафиксировать RAM, FreeRTOS heap и стек.
9. Провести длительный тест.
10. Завершить комментарии, Doxygen и итоговую документацию.
11. Выпустить финальный релиз только после явного подтверждения.
