# Границы базовой прошивки Lorentz

## Назначение

Базовая прошивка Lorentz является демонстрационной и эксплуатационной основой свободнопрограммируемого контроллера LCP2116.

Она должна показать заказчику три независимых механизма:

1. работу модулей системы по внутренней шине X2X;
2. пример циклического опроса внешнего прибора по встроенным линиям RS-485;
3. пример обмена по Ethernet через Modbus TCP.

Проект PNX, IEC 60870-5-104, панель оператора и прикладная логика конкретной установки в эту прошивку не входят.

## 1. Модули X2X

Базовая прошивка поддерживает разработанные модули:

```text
LDO1118
LAI1118
LDI1118
LDI1116
LCT1114
LCT1114_2
```

Конфигурация состава считывается из:

```text
X2X.TXT
```

Порядок строк определяет Modbus slave address модуля.

Для каждого модуля реализуются:

- неблокирующий циклический опрос или запись;
- контроль timeout, CRC, exception и формата ответа;
- фиксация последнего корректного значения;
- отдельная метка качества через состояние связи и ошибки;
- автоматическое восстановление после возврата связи;
- диагностический вывод через команду `x2x`.

Значения при потере связи не обнуляются.

## 2. Пример внешнего прибора FieldSensor

В проект добавляется демонстрационный прибор:

```text
FieldSensor
```

Это не база приборов и не конфигурируемый список сенсоров. Прибор задаётся в исходном коде как понятный пример реализации.

Один и тот же пример запускается независимо на встроенных линиях:

```text
S1
S2
S3
S4
```

Для каждой линии выполняется:

```text
Modbus RTU function: 0x03 Read Holding Registers
start register: 0
register count: 2
poll period: 300 ms
```

Каждая линия должна иметь собственные:

- настройки UART;
- адрес slave;
- машину состояния Modbus master;
- два последних корректных регистра;
- состояние valid/invalid;
- счётчики успешных и ошибочных запросов;
- время последнего корректного обновления.

Пример должен быть явно прокомментирован как шаблон для подключения другого прибора.

Файлы настройки портов считываются совместимо со старой прошивкой в той части, которая относится к аппаратным параметрам линий:

- скорость;
- число бит данных;
- parity;
- stop bits;
- при необходимости режим линии.

Файл со списком датчиков не используется. Тип `FieldSensor`, адреса и карта двух регистров задаются непосредственно в исходном коде.

## 3. Ethernet и Modbus TCP

После завершения X2X и FieldSensor добавляется пример Modbus TCP на Ethernet-порту LCP.

Минимальный пример:

- статический IP считывается из существующего файла сетевых настроек;
- W5500 инициализируется как Modbus TCP server;
- значения `FieldSensor` публикуются в holding registers;
- отдельные status/quality registers показывают валидность и ошибку связи;
- последние значения не заменяются нулём при потере RS-485 связи.

Пример карты:

```text
0      FieldSensor register 0 from S1
1      FieldSensor register 1 from S1
2      quality/status S1
3      FieldSensor register 0 from S2
4      FieldSensor register 1 from S2
5      quality/status S2
...
```

Окончательная карта должна быть зафиксирована в документации после реализации.

## Что намеренно не входит

```text
IEC 60870-5-104
панель оператора
PNX Studio
проектные файлы PNX
база типов внешних приборов
динамическое добавление сенсоров пользователем
технологические алгоритмы конкретной установки
релейные матрицы PNX
газовые таблицы и специализированные драйверы PNX
```

Эти функции относятся к отдельному проекту PNX и не должны усложнять базовый комплект Lorentz.

## Порядок завершения

1. Завершить рефакторинг X2X и повторно проверить существующие модули.
2. Реализовать `FieldSensor` на S1-S4 с периодом 300 мс.
3. Восстановить чтение файлов аппаратных настроек RS-485 из старой прошивки.
4. Добавить отдельную диагностическую команду для FieldSensor.
5. Реализовать Modbus TCP server и публикацию данных FieldSensor.
6. Восстановить чтение IP-настроек Ethernet из старой прошивки.
7. Провести многомодульный и длительный тест.
8. Привести комментарии, Doxygen и руководства в финальный вид.
9. Выпустить финальный релиз базовой прошивки Lorentz.
