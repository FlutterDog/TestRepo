# Обнаруженные дефекты Modbus в прошивках модулей LCT

## Статус

Этот документ фиксирует дефекты, найденные при сверке карт `modbusProceed`.

Драйвер LCP скорректирован под фактические адреса и количество регистров. Перечисленные ниже дефекты находятся внутри прошивок самих модулей и не могут быть надёжно компенсированы на стороне LCP.

## 1. LCT1_r1_b: потеря младших слов RMS

Файл:

```text
00_LCP2116/LCT1_r1_b/main.cpp
```

В `modbusProceed` отсутствует `break` после регистров 2, 4 и 6:

```cpp
case 2:
    regs[i] = convertFloat(params.RMS_A, 0);
    // отсутствует break
case 3:
    regs[i] = convertFloat(params.RMS_A, 1);
    break;
```

То же повторяется для RMS B и RMS C.

Фактическое следствие:

- регистр 2 перезаписывается значением регистра 3;
- регистр 4 перезаписывается значением регистра 5;
- регистр 6 перезаписывается значением регистра 7;
- пары слов RMS A/B/C становятся повреждёнными;
- декодер LCP не может восстановить потерянные младшие слова.

Требуемая коррекция в прошивке LCT1:

```cpp
case 2:
    regs[i] = convertFloat(params.RMS_A, 0);
    break;

case 4:
    regs[i] = convertFloat(params.RMS_B, 0);
    break;

case 6:
    regs[i] = convertFloat(params.RMS_C, 0);
    break;
```

До обновления прошивки модуля первые три значения `float` LCT1 нельзя считать достоверными.

## 2. LCT2_r8_p330: разные коэффициенты для слов одного float

Файл:

```text
00_LCP2116/LCT2_r8_p330/main.cpp
```

Регистр 90 формирует младшее слово тока второго соленоида отключения с коэффициентом `k[SOL_OFF_1]`, а регистр 91 формирует старшее слово с коэффициентом `k[SOL_OFF_2]`.

Фактическое следствие:

- два слова могут быть сформированы из разных чисел `float`;
- значение `solenoid_current_off_2` становится повреждённым, если коэффициенты отличаются.

В обоих вариантах регистра 90 необходимо использовать:

```cpp
k[SOL_OFF_2]
```

то есть тот же коэффициент, который используется в регистре 91.

До обновления прошивки модуля значение `solenoid_current_off_2` следует считать потенциально недостоверным.

## 3. Правило качества данных

Эти ошибки не являются обрывом Modbus: кадр может иметь корректный CRC и корректный размер. Поэтому LCP не способен автоматически определить повреждение отдельных значений.

После исправления прошивок модулей необходимо:

1. пересобрать и прошить соответствующий LCT;
2. проверить контрольные RMS A/B/C для LCT1;
3. проверить `solenoid_current_off_2` для LCT2;
4. сравнить значения с прямыми измерениями или Modbus-анализатором;
5. только после этого считать соответствующие каналы валидными на прикладном уровне.
