# Проверка базовой интеграции FreeRTOS

Статус на 18.07.2026: **аппаратные испытания не выполнены**.

Зафиксированная версия для первого smoke test:

```text
checkpoint/freertos-smoke-test-pending
```

Рабочая ветка дальнейшей интеграции:

```text
feature/freertos-baseline
```

## Назначение проверки

Подтвердить, что переход baseline-прошивки LCP2116 на FreeRTOS не нарушил существующие функции контроллера. На первом этапе вся прежняя логика `setup()/loop()` выполняется внутри одной задачи `LCP`.

## Обязательные проверки checkpoint-версии

1. Конфигурации Debug и Release собираются без ошибок и предупреждений, связанных с FreeRTOS.
2. После прошивки планировщик запускается, а светодиод PLC_ok переключается каждые 500 мс.
3. USB CDC определяется системой и принимает команды `help`, `uptime`, `status`.
4. Значение `uptime_ms` непрерывно увеличивается.
5. USB CDC работает после закрытия и повторного открытия порта.
6. Команда `periodic on` выдаёт статус каждые 10 секунд без зависаний и перезапусков.
7. Watchdog штатно обслуживается базовой задачей.
8. Открытие USB CDC на 1200 baud сохраняет переход в ROM-загрузчик SAM-BA.
9. RS-485, SC16IS7xx, W5500, microSD, батарея и RTC сохраняют прежнее диагностическое поведение.

## Дополнительные проверки рабочей ветки

После успешной проверки checkpoint-версии проверить актуальную `feature/freertos-baseline`:

1. Команда `rtos` возвращает `scheduler=running`.
2. `free_heap_bytes` и `minimum_ever_free_heap_bytes` имеют ненулевые разумные значения.
3. `lcp_stack_free_words` имеет ненулевой запас.
4. Команда `status` включает секцию FreeRTOS.
5. `delay()` из задачи освобождает процессор через `vTaskDelay()`.
6. `delayMicroseconds()` продолжает работать как активная микросекундная задержка.

## Возврат к checkpoint

Для временного перехода к зафиксированной версии:

```bash
git fetch
git switch checkpoint/freertos-smoke-test-pending
```

Для возврата к рабочей ветке:

```bash
git switch feature/freertos-baseline
git pull
```

Checkpoint не следует перемещать или удалять до завершения аппаратных испытаний.
