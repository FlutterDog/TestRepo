# Результаты аппаратных испытаний X2X

## Статус

Дата фиксации: 2026-07-19.

Проверено на контроллере LCP2116 с ATSAM3X8E в ветке `feature/freertos-baseline`.

## 1. Сборка

- Debug: успешно;
- Release: успешно;
- ошибок компиляции и линковки не обнаружено.

## 2. Пустая конфигурация X2X

Файл `X2X.TXT`:

```text
0
Fin
```

Подтверждено:

- конфигурация загружается: `loaded=yes`, `result=ok`, `registry=ok`;
- количество модулей: `modules=0`;
- UART0 возвращается диагностическому echo-test;
- `modbus_state=idle`;
- программный тест CRC: `crc_self_test=ok`;
- `uart_error_count=0`;
- FreeRTOS scheduler работает;
- свободная heap-память стабильна: 22480 байт;
- минимальный свободный стек задачи LCP: 1812 слов;
- USB CDC и watchdog работают без сбоев;
- контроллер допускает штатную повторную прошивку.

## 3. LDI1116

Файл `X2X.TXT`:

```text
1
5
Fin
```

Модуль установлен как X2X slave 1.

Подтверждено:

- UART0 переключается в режим `Modbus RTU master`;
- модуль определяется конфигурацией как `LDI1116`, ID 5;
- циклическое чтение функции Modbus 0x03 работает;
- счётчик `success` непрерывно увеличивается;
- `uart_error_count=0`;
- `communication_error=ok` при исправной линии;
- карта дискретных входов изменяется в соответствии с физическими входами;
- зарегистрированы контрольные значения `digital_inputs=0`, `3` и `145`.

Расшифровка контрольных масок:

```text
3   = биты 0 и 1
145 = биты 0, 4 и 7
```

## 4. Потеря и восстановление связи LDI1116

При разрыве линии получено:

```text
connection=lost
failed=30
consecutive_failures=14
communication_error=timeout
```

После восстановления линии получено:

```text
connection=online
success=2961
failed=48
consecutive_failures=0
communication_error=ok
```

Подтверждено:

- тайм-ауты учитываются как ошибки опроса;
- после пяти последовательных ошибок устанавливается `connection=lost`;
- накопительный счётчик `failed` сохраняет историю ошибок;
- после первого корректного ответа связь автоматически восстанавливается;
- `consecutive_failures` сбрасывается в ноль;
- рабочие данные сохраняют последнее корректное значение до следующего успешного опроса.

## 5. LAI1118

Файл `X2X.TXT`:

```text
1
2
Fin
```

Модуль установлен как X2X slave 1.

Подтверждено:

- модуль определяется конфигурацией как `LAI1118`, ID 2;
- циклическое чтение 17 holding-регистров функцией Modbus 0x03 работает;
- регистр 0 корректно обновляет `digital_inputs`;
- регистры 1..16 корректно декодируются в восемь значений `float`;
- сохранённый порядок слов соответствует прошивке модуля;
- нулевые входы отображаются как `0.00` без `NaN` и бессмысленных значений;
- контрольные аналоговые значения отображались как `12.00` и `14.32` на разных каналах;
- изменение доступности регистров корректно учитывалось как ошибки опроса;
- после восстановления штатного набора регистров связь автоматически возвращалась в состояние `online`;
- `consecutive_failures=0` и `communication_error=ok` после восстановления;
- `uart_error_count=0`.

Зафиксированные контрольные состояния:

```text
float_values=0:12.00, 1:0.00, 2:0.00, ...
float_values=0:12.00, 1:12.00, 2:0.00, ...
float_values=0:12.00, 1:0.00, 2:14.32, ...
```

Накопительный счётчик `failed=178` объясняется намеренным изменением доступного диапазона регистров модуля во время проверки обработки ошибок.

## 6. Вывод

Аппаратный этап для пустой конфигурации, LDI1116 и LAI1118 пройден. Подтверждены:

- загрузка `X2X.TXT`;
- владение UART0;
- Modbus RTU 0x03;
- CRC;
- декодирование 16 дискретных входов;
- декодирование восьми значений `float` LAI1118;
- сохранение требуемого порядка слов `float`;
- обнаружение потери связи;
- автоматическое восстановление связи;
- обработка изменения доступного диапазона регистров;
- совместная работа с FreeRTOS, USB CDC и watchdog.

Остаются непроверенными:

- LDI1118;
- LDO1118;
- LCT1114;
- LCT1114_2;
- несколько модулей в одной цепочке;
- runtime-команды `pause`, `resume` и `reload` на активной линии;
- длительный многомодульный тест.
