# Пример FieldSensor на S1–S4

## Назначение

`FieldSensor` — минимальный демонстрационный драйвер внешнего Modbus RTU-прибора для базовой прошивки Lorentz.

Это не база приборов и не конфигурируемый список датчиков. Тип прибора, адрес slave и карта регистров намеренно заданы в исходном коде. TXT-файлы на microSD изменяют только аппаратные параметры последовательных портов.

Готовые образцы файлов находятся в каталоге:

```text
00_LCP2116/RTOS/sd_card/
```

## Активный пример

Все четыре пользовательских порта работают как независимые Modbus RTU master:

```text
S1 -> slave 1, FC03, holding registers 0..1
S2 -> slave 1, FC03, holding registers 0..1
S3 -> slave 1, FC03, holding registers 0..1
S4 -> slave 1, FC03, holding registers 0..1
```

Общие параметры:

```text
slave address:  1
start register: 0
register count: 2
poll period:    300 ms
timeout:        500 ms
```

Одинаковый адрес допустим, поскольку это четыре независимые физические линии.

## Аппаратное соответствие

```text
S1 -> внешний UART SC16IS7xx
S2 -> ATSAM3X8E Serial1 / HAL UART1
S3 -> ATSAM3X8E Serial3 / HAL UART3
S4 -> ATSAM3X8E Serial2 / HAL UART2
```

Для S1 автоматически определяется один из вариантов:

```text
SC16IS752: UART2_CS, channel B
SC16IS740: UART3_CS, channel A
```

Чётность применяется не только к встроенным UART, но и к S1 через регистр LCR SC16IS7xx.

## Настройки microSD

### BAUD.TXT

Первая значимая строка — количество портов. Далее значения идут в порядке S1, S2, S3, S4.

```text
4
9600
9600
9600
9600
```

Допустимый диапазон в baseline: 300–1000000 бод.

### PARITY.TXT

```text
4
N
N
N
N
```

Допустимые обозначения:

```text
N / NONE / 0 / 78 -> 8N1
O / ODD  / 1 / 79 -> 8O1
E / EVEN / 2 / 69 -> 8E1
```

Числа 78, 79 и 69 являются ASCII-кодами букв N, O и E и поддержаны для совместимости со старыми конфигурациями.

Пустые строки и комментарии `//` или `#` игнорируются.

Если файл отсутствует или содержит ошибку, соответствующая группа параметров сохраняет безопасное значение по умолчанию:

```text
9600 8N1
```

`BAUD.TXT` и `PARITY.TXT` применяются независимо. Например, корректная скорость может быть загружена даже при ошибке в файле чётности.

## Применение настроек

При запуске прошивка ждёт монтирования microSD до двух секунд. Если карта доступна, параметры автоматически загружаются до начала опроса. Если карта недоступна, опрос начинается с 9600 8N1.

Повторное чтение без перезагрузки:

```text
field reload
```

Новые запросы временно не запускаются, активные транзакции завершаются, затем все четыре UART безопасно переинициализируются.

Отчёт:

```text
field
```

В начале отчёта показывается:

```text
serial_config: BAUD.TXT=ok, PARITY.TXT=ok, any_loaded_from_sd=yes
```

Далее для каждого порта выводятся фактически применённые `baud` и `format`.

## Планирование и состояние

Отдельные FreeRTOS-задачи для портов не создаются. Один `field_sensor_service_poll()` обслуживает четыре независимых объекта `ModbusRtuMaster` внутри основной задачи LCP.

Каждый порт имеет собственные:

- transport;
- RX/TX-буферы;
- таймер 300 мс;
- два последних корректных регистра;
- состояние транзакции;
- valid/connection_lost;
- счётчики обмена;
- код exception;
- время последнего корректного обновления.

При единичной ошибке последние корректные значения сохраняются. После пяти последовательных ошибок:

```text
valid=no
connection=lost
```

После первого корректного ответа связь автоматически восстанавливается.

## Владение UART

При запуске FieldSensor:

- echo-test отключается на S1–S4;
- echo PC и HMI на других SC16IS-каналах остаётся;
- X2X независимо использует UART0;
- один half-duplex-порт не используется одновременно несколькими protocol engine.

## Service console

```text
field
field reload
field pause
field resume
```

`field pause` не обрывает уже начатый RTU-кадр.

## Изменение примера прибора

Основная конфигурация находится в:

```text
app/field/field_sensor_service.cpp
```

Файлы настроек портов:

```text
app/field/field_serial_config.cpp
app/field/field_serial_config.hpp
```

Для нового прибора разработчик может использовать существующий `ModbusRtuMaster`, изменить slave address и карту регистров, затем добавить явное декодирование результата. Динамическая база приборов для базовой прошивки не требуется.

## Альтернативный slave-режим

Минимальный reusable slave FC03 расположен в:

```text
protocol/modbus_rtu/modbus_rtu_slave.cpp
protocol/modbus_rtu/modbus_rtu_slave.hpp
```

Отключённый пример подключения двух holding-регистров:

```text
app/field/field_sensor_slave_example.cpp
```

Рабочая baseline-конфигурация оставляет все S1–S4 в режиме master.

## Проверка

1. Скопировать `BAUD.TXT` и `PARITY.TXT` из `RTOS/sd_card` в корень microSD.
2. Настроить внешний slave address 1 на параметры соответствующего порта.
3. Предоставить holding-регистры 0 и 1.
4. Выполнить `field`.
5. Проверить `BAUD.TXT=ok`, `PARITY.TXT=ok`, `valid=yes`, `connection=online` и рост `success`.
6. Разорвать линию и проверить потерю связи после пяти ошибок без обнуления значений.
7. Восстановить линию и проверить автоматическое восстановление.
