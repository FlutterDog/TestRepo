# Stage 5 — QC + Release Build

## Purpose

Stage five выполняет финальную проверку качества и собирает релиз.

Это итеративная стадия clean-up и release build: она может выполняться повторно, пока не будет получен чистый `board_guide.md`.

Stage five не должен переносить в релиз служебные следы authoring-процесса.

---

## QC focus

Финальная проверка должна включать не только factual cleanliness, но и explanatory adequacy.

### Обязательно проверить

* текст не сухой и не сведён к инвентарным карточкам;
* в каждой крупной главе есть назначение, реализация, характеристики, возможности и ограничения;
* `author_notes.md` и official references использованы там, где ими можно усилить слабые разделы;
* нет неподтверждённых обещаний;
* нет process-language;
* figure links и captions согласованы;
* релиз объясняет плату как платформу и как конкретную аппаратную реализацию.

---

## Outputs

* `qc_report.md`
* `docs/release/board_guide.md`
* `docs/release/IMG/`
* при необходимости: `draft_v1.md`

---

## Rule of completion

Stage five считается завершённым, только если релизный файл уже чистый.

Если после прохода текст ещё не дотягивает до clean release, стадия должна честно вернуть `draft_v1.md` и `qc_report.md` для следующего прохода.
