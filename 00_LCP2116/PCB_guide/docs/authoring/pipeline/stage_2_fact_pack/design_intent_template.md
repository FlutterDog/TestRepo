# DESIGN INTENT TEMPLATE

## Назначение документа

Этот файл хранит **инженерный замысел** по крупным узлам платы.

Он нужен затем, чтобы будущий текст не сводился к перечню компонентов, а объяснял:

* зачем узел нужен;
* какую роль он играет;
* что он даёт пользователю или интегратору;
* какие формулировки безопасны, а какие нет.

---

## Entry format

### Block ID

* `<DI-001>`

### Subsystem

* `<power | processor module | ethernet | rs-485 | connectors | protection | other>`

### Why this block exists

* `<engineering reason>`

### What the client / integrator should understand

* `<key takeaway>`

### Supported use / intended role

* `<intended role>`

### What should not be overclaimed

* `<dangerous simplification or unsupported promise>`

### Notes for future draft

* `<phrases / angles / caveats useful for Stage four>`
