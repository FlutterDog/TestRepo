# STAGE PROMPT — Stage 1 (Freeze Sources)

> ℹ️ **Назначение документа**
> Этот stage prompt задаёт **рабочий режим генерации** для Stage 1.
>
> // ЭТО НЕ README
> // Его задача — не объяснить процесс подробно,
> // а жёстко задать правила, формат результата и quality bar.

Ты работаешь по канону репозитория:

* `docs/authoring/pipeline/PIPELINE.md`
* `/README.md`

Сейчас активная стадия: **Stage 1 — Freeze Sources**.

## GOAL

Выполнить Stage 1 как freeze-комплект:

* собрать один комплект инженерных исходников;
* разложить его строго по `sources/*`;
* заполнить `docs/authoring/artifacts/stage_1_freeze_sources/release_manifest.md`;
* начать `docs/authoring/artifacts/stage_1_freeze_sources/file_map.md`.

---

## HARD RULES (не нарушать)

### 1) Stage 1 не пишет Board Guide

На этой стадии запрещено:

* писать текст документа;
* анализировать схему глубже, чем нужно для freeze и стартовой трассируемости;
* генерировать narrative по узлам.

### 2) Никаких выводов по памяти

Разрешено использовать только:

* файлы из freeze-комплекта;
* структуру репозитория;
* канон пайплайна.

### 3) После Stage 1 `sources/*` считаются замороженными

После фиксации Stage 1 исходники в `sources/*` больше не редактируются.

Если комплект источников всё же меняется:

* это должно быть осознанным действием;
* обязательно обновляется `release_manifest.md`;
* явно фиксируется, что изменилось и почему.

### 4) Stage 1 не ведёт `open_items.md`

На Stage 1 допустим `UNKNOWN` в `file_map.md`.
Но `open_items.md` начинается только со Stage 2.

### 5) Не смешивать `sources/extras/` и `artifacts/IMG/`

* `sources/extras/` — часть Source of Truth;
* `docs/authoring/artifacts/IMG/` — изображения для будущего документа.

### 6) Один комплект — одна версия платы

Если есть признаки смешивания ревизий:

* это нужно явно указать как риск;
* freeze нельзя считать чистым, пока комплект не приведён к одной версии.

---

## FREEZE CONTROL BAR (обязательное качество результата)

> ℹ️ **Главный смысл этого блока**
> Stage 1 должен не просто перечислить файлы, а привести репозиторий к режиму:
> один комплект источников, один SoT, одна карта трассируемости верхнего уровня.

### Главное правило

Результат Stage 1 должен отвечать на три вопроса:

* что именно считается Source of Truth;
* где именно это лежит;
* нет ли второй правды или смешивания версий.

### Что запрещено

Запрещены:

* расплывчатые формулировки без проверки состава комплекта;
* формальный `release_manifest.md`, который не соответствует реальному `sources/`;
* пустой или почти пустой `file_map.md`;
* выводы о составе комплекта без опоры на фактические файлы.

### Как выглядит хороший результат Stage 1

Хороший результат Stage 1:

* чётко показывает, что есть, чего не хватает и где есть риск смешивания версий;
* приводит каноническую раскладку по `sources/*`;
* даёт рабочий черновик `release_manifest.md`;
* запускает `file_map.md` с верхнеуровневыми утверждениями и якорями.

### Допустимая степень неполноты

Допустимо:

* иметь `UNKNOWN` в `file_map.md`;
* не покрыть всю схему целиком;
* не знать часть доказательств на верхнем уровне.

Недопустимо:

* считать freeze завершённым при явном смешивании версий;
* оставлять источники вне `sources/`;
* не начать трассируемость вообще.

## OUTPUT FORMAT

> ℹ️ **Основной результат Stage 1**
> Stage 1 должен возвращать результат блоками, чтобы пользователь видел:
> риски комплекта, раскладку, freeze-паспорт и старт трассируемости.

Всегда возвращай результат блоками в таком порядке:

1. Проверка комплекта:

   * что есть;
   * чего не хватает;
   * есть ли признаки смешивания версий.

2. Раскладка по `sources/*`:

   * если это актуально;
   * если раскладка уже сделана правильно, так и сказать.

3. `## docs/authoring/artifacts/stage_1_freeze_sources/release_manifest.md`

   * черновик заполнения.

4. `## docs/authoring/artifacts/stage_1_freeze_sources/file_map.md`

   * стартовые записи: от пяти до пятнадцати утверждений.

Без:

* Board Guide текста;
* open items;
* narrative по схеме.

---

## QUALITY BAR (самопроверка)

* Ясно, относится ли комплект к одной версии платы.
* Ясно, чего не хватает для чистого freeze.
* `release_manifest.md` отражает реальный состав `sources/*`.
* `file_map.md` содержит стартовую трассируемость верхнего уровня.
* Нигде не смешаны доказательства и картинки для документа.
* Если результат выглядит как «формальная отписка по папкам», он не проходит quality bar.
