# README — chapter-by-chapter pipeline for Board Guide

Этот процесс используется для подготовки технического описания платы по главам.  
Цель процесса — не писать большой текст сразу, а собирать документ по узлам и главам, с контролем качества на каждом шаге.

---

## Общий принцип

Работа идёт по схеме:

1. создать outline;
2. разложить chapter packs;
3. уточнить outline и покрытие источниками;
4. написать draft по одной главе;
5. довести каждую главу отдельно;
6. собрать общий Markdown и выполнить финальную чистку.

Распределение ролей:

- **Автор** задаёт архитектуру документа, готовит и сортирует исходные материалы, проверяет смысл и инженерную точность.
- **ChatGPT** помогает уточнить структуру, пишет главы по готовым материалам, сжимает текст, убирает повторы и выравнивает стиль.

---

## Stage 1 — Outline

### Цель
Создать скелет документа: главы, подглавы, порядок раскрытия темы.

### Действия автора
- определить список глав;
- определить порядок глав;
- дать краткое назначение каждой главы;
- отметить главы, для которых уже есть материалы;
- отметить главы, для которых материалов пока нет.

### Действия ChatGPT
- проверить логичность структуры;
- предложить укрупнение, дробление или перестановку глав;
- убрать дубли между главами;
- сделать чистовую версию outline в Markdown.

### Вход
- список предполагаемых глав;
- краткие комментарии автора по смыслу документа.

### Выход
- `outline.md`

---

## Stage 2 — Chapter packs

### Цель
Собрать материалы отдельно по каждой главе.

### Правило
Материалы раскладываются **по главам**, а не общей кучей.

### Рекомендуемая структура
```text
sources/
  01_intro/
  02_architecture/
  03_power/
  04_cpu/
  05_ethernet/
  06_rs485/
  ...
```

### Внутри каждой папки
```text
schematic/
bom/
datasheet_excerpt/
manual_excerpt/
author_notes/
images/
```

### Действия автора
- создать папки по главам;
- положить в каждую папку только относящиеся к главе материалы;
- добавить:
  - фрагменты схемы;
  - BOM по узлу или по листу;
  - excerpts из datasheet / manual;
  - краткие author notes;
  - картинки, если нужны;
- не класть большие несортированные архивы.

### Действия ChatGPT
- проверить, хватает ли материалов для написания главы;
- указать, где chapter pack пустой или слабый;
- указать, где материалов слишком много и их надо сузить.

### Вход
- папки `sources/<chapter>/...`

### Выход
- chapter packs по всем главам.

---

## Stage 3 — Уточнение outline и coverage check

### Цель
Проверить, что каждая глава имеет источники и понятные границы.

### Действия автора
- загрузить актуальный outline и chapter packs;
- ответить на вопросы по спорным местам;
- при необходимости добавить недостающие файлы.

### Действия ChatGPT
- проверить покрытие outline источниками;
- отметить:
  - какие главы готовы к draft;
  - где не хватает материалов;
  - где есть пересечение двух глав;
- уточнить формулировки заголовков и подзаголовков;
- выдать финальный outline для drafting stage.

### Вход
- `outline.md`
- chapter packs

### Выход
- `outline_locked.md`
- список замечаний по покрытию источниками

---

## Stage 4 — Draft по одной главе

### Цель
Написать одну главу на основе одного chapter pack.

### Жёсткое правило
За один проход пишется **одна глава**, а не весь документ.

### Действия автора
- выбрать одну главу;
- загрузить материалы только по этой главе;
- при необходимости дать короткий комментарий:
  - что важно подчеркнуть;
  - чего нельзя утверждать;
  - какой нужен стиль.

### Действия ChatGPT
- прочитать chapter pack;
- написать цельный технический текст главы;
- писать в плотном manual-style;
- не писать осторожные фразы типа:
  - «возможно»,
  - «вероятно»,
  - «по-видимому»;
- не добавлять мусорные связки;
- не выходить за scope главы.

### Рекомендуемая команда
```text
Напиши draft только для этой главы.
Используй только приложенные материалы chapter pack.
Стиль: technical manual / TI Feature Description for board guide.
Без осторожных формулировок.
Без списков в итоговом тексте.
Верни готовый Markdown.
```

### Вход
- материалы одной главы

### Выход
- `chapter_<name>_draft.md`

---

## Stage 5 — Доводка по одной главе

### Цель
Сделать текст плотным, чистым и годным для сборки в общий документ.

### Действия автора
- прочитать draft;
- отметить:
  - фактические ошибки;
  - слабые места;
  - повторы;
  - лишние слова;
  - места для figure;
- при необходимости прислать правки прямо в тексте.

### Действия ChatGPT
- ужать текст;
- убрать дубли;
- усилить технический стиль;
- убрать разговорные слова;
- привести терминологию к единому виду;
- вставить placeholders для figure по запросу автора.

### Рекомендуемая команда
```text
Доведи эту главу.
Убери дубли, мусорные слова и рыхлые формулировки.
Сохрани технический стиль.
Не расширяй scope.
Не перепридумывай архитектуру.
Верни готовый Markdown.
```

### Вход
- `chapter_<name>_draft.md`
- замечания автора

### Выход
- `chapter_<name>_final.md`

---

## Stage 6 — Сборка общего Markdown

### Цель
Собрать все готовые главы в один документ и выполнить один финальный проход.

### Действия автора
- собрать все `chapter_<name>_final.md` в один файл;
- вставить figure, если они уже готовы;
- проверить порядок глав;
- добавить front matter, title, contents, если нужно.

### Действия ChatGPT
- проверить общий документ на:
  - повторы между главами;
  - скачки логики;
  - несогласованность терминов;
  - различия в стиле;
- предложить минимальные правки;
- выполнить один общий чистовой проход.

### Важное ограничение
Финальный проход по всему документу выполняется **один раз** или максимум **два раза**.  
Не использовать большой финальный проход как замену по-главной доводке.

### Вход
- `board_guide_full.md`

### Выход
- `board_guide_release.md`

---

## Рабочая модель взаимодействия

Для каждой главы используется один и тот же цикл:

1. автор выбирает главу;
2. автор загружает chapter pack;
3. ChatGPT пишет draft;
4. автор читает и правит;
5. ChatGPT доводит текст;
6. глава фиксируется;
7. переход к следующей главе.

---

## Что не делать

- не загружать весь проект одной кучей;
- не просить написать сразу весь документ;
- не использовать полный datasheet, если нужен только summary и protection features;
- не делать большой global fact pack;
- не надеяться на большой финальный проход как на основное средство правки.

---

## Что делать сейчас

1. утвердить `outline.md`;
2. создать папки chapter packs;
3. наполнить chapter pack для первой главы;
4. загрузить первую главу в работу;
5. пройти cycle draft → review → final;
6. повторить для следующей главы.
